# Add rate limiting middleware

No rate limiting exists on any endpoint. Combined with lack of auth (separate concern), any client can flood the server.

## Implementation
- Add tower::limit::RateLimitLayer or tower_governor to Axum router
- Configure via RATE_LIMIT_RPS env var (requests per second, default: 100)
- Apply per-IP rate limiting
- Return HTTP 429 Too Many Requests when exceeded
- Add rate limit headers (X-RateLimit-Limit, X-RateLimit-Remaining)
- Consider separate limits for read (GET /ls) vs write (POST /kill-*) operations
