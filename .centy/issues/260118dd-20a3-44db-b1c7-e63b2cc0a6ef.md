# README project structure and proto description are outdated

## Problem

The README has several inaccuracies about the project structure and architecture:

1. **Project structure shows `src/tmux/`** but the domain logic is now in `crates/tmux-gateway-core/`. The `crates/` directory is not mentioned at all.

1. **Claims proto is “hand-maintained source of truth”** — `schemas/tmux_gateway.proto` says it’s “hand-maintained” in the project structure section, but the proto is actually code-generated from Rust macros in `src/api/grpc/messages.rs` and `src/api/grpc/server.rs`. It’s code-first gRPC, not schema-first.

1. **Shows `build.rs`** in the project structure but there is no build.rs — the project uses code-first proto generation via macros, not a build script.

1. **Architecture diagram** shows REST and GraphQL going through gRPC, which is misleading — they share the same domain core but don’t route through gRPC.

Note: Issue #12 covers wrong endpoint/method names but doesn’t address the structural inaccuracies.
