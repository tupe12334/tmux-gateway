# Add CI schema drift detection and Docker build

The CI pipeline only builds and tests but misses two critical checks.

## Schema Drift Detection

Add a CI step that:

1. Runs make build (which exports schemas)
1. Checks git diff on schemas/ directory
1. Fails if schemas have drifted from committed versions
   This ensures schema changes are always committed and reviewed in PRs.

## Docker Build in CI

* Add a step to build the Docker image in CI (build only, no push)
* Ensures Dockerfile stays working and dependencies resolve
* Consider adding ghcr.io push on tag/release

## Clippy in CI

* The pre-push hook runs clippy but CI does not
* Add cargo clippy –workspace – -D warnings to ci.yml
