# Dockerfile: run as non-root user and add HEALTHCHECK

The Dockerfile has two security/operational best practice gaps.

## Run as non-root user:
The container runs as root by default (no USER directive in runtime stage). This is a security risk - if the application is compromised, the attacker has root access in the container.

### Fix:
```dockerfile
RUN useradd -r -s /bin/false tmux-gateway
USER tmux-gateway
```

## Add HEALTHCHECK:
No HEALTHCHECK instruction means container orchestrators (Docker Compose, k8s via Docker) can't auto-detect unhealthy containers.

### Fix:
```dockerfile
HEALTHCHECK --interval=30s --timeout=5s --retries=3   CMD curl -f http://localhost:${HTTP_PORT}/health || exit 1
```

Note: curl is already installed in the runtime stage (line 20).
