# Add respawn_pane domain operation to restart a pane's process

When a command in a pane exits, the pane becomes dead. Tmux respawn-pane restarts it with the same or a new command. The gateway has no way to restart a dead pane without killing and recreating it.

## Problem
- A pane running a process that exits leaves a dead pane
- The only recovery is kill_pane + split_window, which changes the layout
- tmux respawn-pane -t target [-c start_dir] [command] restarts in place
- Automation workflows need to restart failed processes without layout disruption

## What to do
Add to tmux-gateway-core:

pub fn respawn_pane(target: &str, command: Option<String>) -> Result<(), TmuxError>

- If command is None, restarts with the original command
- If command is Some, starts the new command instead
- Validate target with validate_pane_target
- Validate command for injection

## Acceptance criteria
- Dead panes can be restarted in place
- Optional new command can be specified
- All API layers can expose this operation
