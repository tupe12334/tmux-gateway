# Enforce minimum test coverage threshold in CI

CI generates coverage reports via cargo-tarpaulin but does not enforce a minimum coverage percentage. Coverage can silently regress as new code is added without tests.

## What to do
- Add --fail-under flag to the cargo tarpaulin CI step (e.g., --fail-under 60)
- Gradually increase the threshold as coverage improves
- Add a coverage badge to the README

## Acceptance criteria
- CI fails if test coverage drops below the configured threshold
- Coverage percentage is visible in PR checks or comments
