# Add timeout for tmux operations to prevent thread pool exhaustion

All core tmux operations use tokio::task::spawn_blocking() with no timeout. If tmux hangs (e.g. waiting for user input, broken pipe), the blocking thread is held indefinitely.

## Problem
- spawn_blocking() threads are limited (default: 512 in tokio)
- A hanging tmux command consumes one thread forever
- Enough hanging commands could exhaust the entire blocking thread pool
- All subsequent tmux operations would queue indefinitely

## Fix
- Wrap each spawn_blocking call in tokio::time::timeout()
- Add TMUX_COMMAND_TIMEOUT_SECS env var (default: 30)
- Return TmuxError::Timeout when exceeded
- Map to appropriate HTTP 504 / gRPC DEADLINE_EXCEEDED status codes
