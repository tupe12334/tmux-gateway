# Add command alias management domain operations

## Problem

Tmux supports user-defined command aliases (`command-alias` server option) that map short names to full commands. This is a server-level configuration feature that power users rely on for workflow customization. The gateway has no domain operations for managing these aliases, meaning:

1. Users cannot discover what aliases are configured on a tmux server
2. Programmatic alias management (adding aliases for automation scripts) is not possible
3. The gateway cannot warn clients when a command might be aliased

## Proposed change

Add command alias domain operations:

```rust
/// A tmux command alias mapping.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct CommandAlias {
    pub name: String,
    pub command: String,
}

/// List all command aliases configured on the server.
pub async fn list_command_aliases(
    executor: &impl TmuxExecutor,
) -> Result<Vec<CommandAlias>, TmuxError>;

/// Add or update a command alias.
pub async fn set_command_alias(
    executor: &impl TmuxExecutor,
    name: &str,
    command: &str,
) -> Result<(), TmuxError>;

/// Remove a command alias.
pub async fn remove_command_alias(
    executor: &impl TmuxExecutor,
    name: &str,
) -> Result<(), TmuxError>;
```

Implementation uses `tmux show-options -s command-alias` to list and `tmux set-option -s command-alias` to modify.

## Why this matters

- Enables programmatic tmux server configuration management
- `CommandAlias` is a simple value object — clean domain model
- Important for introspection: knowing what aliases exist helps debug unexpected behavior
- Complements tmux option management (#102) with this specific high-use option
- API-agnostic: configuration management is equally useful from REST, GraphQL, and gRPC

## Acceptance criteria

- [ ] `CommandAlias` value object with name and command
- [ ] `list_command_aliases` parses tmux show-options output
- [ ] `set_command_alias` adds/updates an alias
- [ ] `remove_command_alias` deletes an alias
- [ ] Name validation (no spaces, shell metacharacters)
- [ ] Unit tests with mock executor
- [ ] Handle empty alias list (no aliases configured)
