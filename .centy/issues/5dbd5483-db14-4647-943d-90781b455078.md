# Add link-window and unlink-window domain operations for window sharing

tmux supports linking a window from one session into another session, allowing the same window to appear in multiple sessions simultaneously. This is a powerful feature for collaborative and multi-context workflows that has no domain representation.

## Problem
- No way to share a window between sessions through the gateway
- Users who want the same terminal view in multiple session contexts must duplicate work
- Missing a core tmux capability that enables advanced session management patterns

## Proposed solution
Add domain operations for window linking:

\`\`\`rust
/// Link an existing window from a source session into a target session.
pub async fn link_window(
    source: &str,      // source window target, e.g. "session1:0"
    destination: &str,  // destination session target, e.g. "session2"
) -> Result<(), TmuxError>

/// Unlink (remove) a window from a session. If the window is linked
/// to other sessions, it remains in those sessions. If it is the last
/// link, this is equivalent to killing the window.
pub async fn unlink_window(
    target: &str,  // window target, e.g. "session:0"
) -> Result<(), TmuxError>
\`\`\`

Both operations should:
- Validate targets using existing validation functions
- Map tmux errors through \`TmuxError::from_stderr\`
- Follow the same spawn_blocking pattern as other domain operations

## Scope
- New modules: \`link_window.rs\`, \`unlink_window.rs\`
- Add to \`TmuxCommands\` trait
- Export from \`lib.rs\`
- Add integration tests
