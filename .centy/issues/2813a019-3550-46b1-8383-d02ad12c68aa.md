# Add tmux server socket support for multi-server domain operations

The domain assumes a single default tmux server. Tmux supports multiple servers via the -L (socket name) and -S (socket path) flags. The gateway cannot manage sessions across different tmux server instances.

## Problem
- All domain operations implicitly use the default tmux server socket
- Production environments may run multiple tmux servers for isolation
- No way to target a specific server when listing sessions, creating sessions, etc.
- tmux -L work and tmux -L staging can coexist — the gateway sees only one

## What to do
Add an optional server context to domain operations:

pub struct TmuxServer {
    pub socket: Option<ServerSocket>,
}

pub enum ServerSocket {
    Name(String),    // -L flag
    Path(PathBuf),   // -S flag
}

Either thread TmuxServer through domain functions or make it a configuration that the backend accepts. The default (None) preserves current behavior.

## Acceptance criteria
- Domain operations can target a specific tmux server socket
- Default behavior (no socket specified) is unchanged
- All three API layers can specify the target server
