# Warn or fail on invalid CORS_ORIGINS values instead of silently dropping

In src/main.rs, CORS origins are parsed with filter_map(|o| o.parse().ok()), silently dropping invalid origins. If all origins fail to parse, the server runs with an empty CORS allowlist without any warning.

## Problem:

* No log warning when an origin fails to parse
* No startup error if ALL origins are invalid (effectively disabling CORS silently)
* Users think CORS is configured when it’s actually not working

## Solution:

* Log a warning for each unparseable origin
* Optionally fail at startup if no valid origins remain after parsing
* Log the effective CORS configuration at info level during startup
