# Add find-window domain operation for text search across panes

There is no domain operation for searching across tmux windows and panes by content or name. The tmux \`find-window\` command supports searching window names, titles, and visible pane content — a fundamental capability for any tmux management tool.

## Problem
- To find which pane contains specific output, callers must list all windows/panes and capture each individually
- No way to search by window name pattern at the domain level
- Missing a core tmux feature that enables efficient navigation and discovery

## Proposed solution
Add a \`find_window\` domain operation:

\`\`\`rust
pub struct FindResult {
    pub session: String,
    pub window_index: u32,
    pub window_name: String,
    pub pane_id: String,
    pub match_context: String,   // the matching line or name
}

pub async fn find_window(
    session: &str,
    pattern: &str,
) -> Result<Vec<FindResult>, TmuxError>
\`\`\`

- Searches window names, titles, and visible pane content within a session
- Returns structured results with enough context to navigate to the match
- Validates session target and search pattern (non-empty, safe characters)
- Pattern validation should prevent command injection while allowing reasonable search terms

## Scope
- New \`find_window.rs\` module in \`tmux-gateway-core\`
- New \`FindResult\` domain model type
- Add to \`TmuxCommands\` trait
- Add pattern validation to \`validation.rs\`
