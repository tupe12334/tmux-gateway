# Add session topology pure functions for aggregate statistics

## Problem

There are no domain-level functions for computing aggregate statistics across sessions. Common questions like "how many total panes are running?", "what's the largest session?", or "how much terminal area is in use?" require API consumers to manually aggregate data from multiple list calls.

These computations are pure functions over already-fetched domain data — perfect candidates for the functional core.

## Proposed change

Add a `topology` module in `tmux-gateway-core` with pure aggregate functions:

```rust
/// Summary statistics for a session
pub struct SessionStats {
    pub name: String,
    pub window_count: u32,
    pub total_pane_count: u32,
    pub total_area: u64,          // sum of all pane areas
    pub has_active_window: bool,
}

/// Summary statistics for the entire tmux server
pub struct ServerStats {
    pub session_count: usize,
    pub total_windows: u32,
    pub total_panes: u32,
    pub attached_session_count: usize,
    pub detached_session_count: usize,
}

// Pure functions
pub fn compute_session_stats(
    session: &TmuxSession,
    windows: &[TmuxWindow],
    panes_by_window: &[(u32, Vec<TmuxPane>)],
) -> SessionStats;

pub fn compute_server_stats(sessions: &[TmuxSession]) -> ServerStats;

pub fn largest_session(sessions: &[TmuxSession]) -> Option<&TmuxSession>;

pub fn smallest_session(sessions: &[TmuxSession]) -> Option<&TmuxSession>;

pub fn total_pane_count(sessions: &[TmuxSession]) -> u32;

pub fn sessions_summary(sessions: &[TmuxSession]) -> String;
```

## Why this matters

- Health checks (#111) need aggregate stats to assess server health
- Resource limit policies (#114) need totals to enforce limits
- Dashboard/monitoring UIs need summary data without multiple API calls
- All functions are pure — trivially testable, no tmux dependency
- API layers can expose a `/stats` or `serverStats` query backed by these pure functions

## Acceptance criteria

- [ ] `SessionStats` and `ServerStats` structs with meaningful fields
- [ ] Pure functions for computing session-level and server-level statistics
- [ ] `largest_session` and `smallest_session` by window count
- [ ] `total_pane_count` aggregation across all sessions
- [ ] Unit tests with various topologies (single session, many sessions, empty)
- [ ] No I/O — all functions operate on already-fetched domain data
