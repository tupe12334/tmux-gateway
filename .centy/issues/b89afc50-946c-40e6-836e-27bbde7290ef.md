# Add declarative session specification value object for template-based creation

## Problem

Issue #46 adds imperative composite operations like `create_session_with_windows()`. But the domain lacks a **declarative specification pattern** — a value object that describes the desired session layout, which the domain can then realize into actual tmux state.

This is a distinct DDD concept: #46 is imperative ("do A then B then C"), while a specification is declarative ("here is the desired end state, make it happen"). Specifications enable:
- Reusable session templates (dev environments, monitoring setups)
- State reconciliation (compare current state to spec, apply differences)
- Serializable session definitions (JSON/YAML template files)

## Proposed changes

1. Add specification value objects:
```rust
pub struct SessionSpec {
    pub name: String,
    pub windows: Vec<WindowSpec>,
    pub environment: HashMap<String, String>,
}

pub struct WindowSpec {
    pub name: String,
    pub command: Option<String>,
    pub panes: Vec<PaneSpec>,
}

pub struct PaneSpec {
    pub command: Option<String>,
    pub split: SplitDirection,  // Horizontal | Vertical
}
```

2. Add domain operation:
```rust
pub async fn apply_session_spec(spec: &SessionSpec) -> Result<SessionDetail, TmuxError>
```

3. Spec is a pure value object — serializable, comparable, testable without tmux
4. `apply_session_spec` composes atomic operations internally, handling partial failure rollback

## Relationship to other issues

- #46: Imperative composites — this issue adds the declarative layer on top
- #74: SessionDetail aggregate — the return type from apply
- #76: Initial commands — specs use this for pane/window commands
- #77: Environment variables — specs include env var setup

## Acceptance criteria

- [ ] SessionSpec, WindowSpec, PaneSpec value objects exist in core crate
- [ ] apply_session_spec realizes a spec into actual tmux state
- [ ] Partial failure during apply attempts rollback (best-effort cleanup)
- [ ] Specs are pure data: PartialEq, Clone, Debug
- [ ] Serializable with serde for template file support
