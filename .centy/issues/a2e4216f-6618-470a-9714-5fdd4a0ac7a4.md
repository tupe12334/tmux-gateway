# Add WebSocket support for real-time pane output streaming

Currently there is no way to stream real-time output from tmux panes. This is critical for building interactive terminal UIs on top of the gateway.

## Requirements
- WebSocket endpoint at /ws/pane/{target} that streams pane output in real-time
- GraphQL subscription for pane output changes
- gRPC server-streaming RPC for pane output
- Configurable polling interval or use tmux pipe-pane for push-based updates

## Implementation
- Use axum's WebSocket support (already included via axum features)
- Add async-graphql Subscription type
- Add server-streaming RPC to proto definition
- Use tmux pipe-pane or periodic capture-pane for output sourcing
- Handle client disconnect cleanup (stop piping when no subscribers)
