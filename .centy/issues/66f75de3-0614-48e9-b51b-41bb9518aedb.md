# Add window name validation to domain validation module

## Problem
The validation module (validation.rs) has validate_session_name, validate_session_target, validate_window_target, and validate_pane_target — but there is no validate_window_name function. The new_window and rename_window operations accept arbitrary window names without domain-level validation.

## Current state
- validate_session_name exists and checks length, emptiness, and forbidden characters
- Window names go through no validation before being passed to tmux
- If tmux rejects the name, the error comes back as a raw CommandFailed with tmux stderr

## Proposed change
Add `validate_window_name(name: &str) -> Result<(), ValidationError>` to the validation module. Window names in tmux have similar constraints to session names (no dots, colons, or control characters). Call this validation from new_window and rename_window before executing the tmux command.

## Acceptance criteria
- [ ] validate_window_name function added to validation.rs
- [ ] Validates: non-empty, reasonable length, no forbidden characters
- [ ] Called from new_window and rename_window operations
- [ ] Unit tests cover valid and invalid window names
