# Add a domain-level tmux server health check function

There is no domain function to check whether the tmux server is running and healthy. The only way to discover this is by calling list_sessions and checking if it fails with TmuxNotRunning.

## Problem
- Health check logic is duplicated: preflight.rs checks tmux -V at startup, gRPC health service checks separately, REST /health doesn't check tmux at all
- No single domain function answers "is tmux available and responding?"
- list_sessions masks server-down as empty list (stderr "no server running" returns Ok(vec![]))
- Issue #25 asks for gRPC health check to verify tmux — but the solution should be a domain function, not gRPC-specific

## What to do
Add a domain health check function in the core crate:

pub async fn check_tmux_health() -> Result<TmuxHealth, TmuxError>

pub struct TmuxHealth {
    pub version: String,
    pub server_running: bool,
    pub session_count: u32,
}

All API health endpoints and gRPC health services can delegate to this single function.

## Acceptance criteria
- A single domain function reports tmux health status
- All API layers use this function for their health checks
- The function distinguishes "tmux not installed" from "tmux installed but server not running"
