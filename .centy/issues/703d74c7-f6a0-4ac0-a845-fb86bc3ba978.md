# Add select_window and select_pane domain operations for focus management

The domain can create, list, kill, and rename windows and panes but cannot change which window or pane is active. Tmux select-window and select-pane are core navigation operations missing from the gateway.

## Problem
- TmuxWindow and TmuxPane have an active: bool field, but there is no operation to change it
- Clients cannot programmatically switch focus between windows/panes
- Automation workflows that need to interact with specific panes must manually send tmux hotkeys

## What to do
Add to tmux-gateway-core:

pub async fn select_window(target: &str) -> Result<(), TmuxError>
pub async fn select_pane(target: &str) -> Result<(), TmuxError>

- Validate targets with appropriate validators
- Map to tmux select-window -t and select-pane -t

## Acceptance criteria
- Active window and pane can be changed via domain operations
- Target validation is applied before execution
- All three API layers can expose these operations
