# Add session snapshot and restore domain operations

There is no way to capture a session's full state (window layout, pane arrangement, working directories, running commands) and restore it later. This is essential for session persistence across tmux server restarts.

## Problem
- Killing the tmux server or rebooting the machine loses all session state
- No domain operation to serialize a session's structure into a portable format
- No domain operation to recreate a session from a saved snapshot
- Users must manually rebuild sessions after restarts

## What to do
Add to tmux-gateway-core:

pub struct SessionSnapshot {
    pub name: String,
    pub windows: Vec<WindowSnapshot>,
}

pub struct WindowSnapshot {
    pub name: String,
    pub layout: String,          // tmux layout string
    pub panes: Vec<PaneSnapshot>,
}

pub struct PaneSnapshot {
    pub working_directory: String,
    pub command: Option<String>,  // running command, if detectable
}

pub fn snapshot_session(name: &str) -> Result<SessionSnapshot, TmuxError>
pub fn restore_session(snapshot: SessionSnapshot) -> Result<String, TmuxError>

- snapshot_session queries list-windows and list-panes with format strings for layout, path, and command
- restore_session creates session + windows + panes with the saved directories and commands
- Snapshots are plain data (serializable) — API layers can expose as JSON/protobuf

## Acceptance criteria
- A session's structure can be captured as a serializable snapshot
- A snapshot can recreate the session with the same layout and working directories
- All API layers can expose snapshot and restore operations
