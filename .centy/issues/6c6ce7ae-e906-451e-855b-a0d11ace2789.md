# Add domain retry policy for transient tmux command failures

tmux commands can fail transiently due to socket contention, server restarts, or temporary resource pressure. The domain has no concept of retryable vs permanent failures, forcing all error handling to the API layers or clients.

## Problem

- tmux socket contention can cause transient `CommandFailed` errors that succeed on retry
- Server restart during an operation causes a failure that is retryable moments later
- The `TmuxError` enum doesn't distinguish transient from permanent failures
- Clients must implement their own retry logic without domain guidance
- Issue #87 (timeout) covers deadline but not retry-after-failure
- Issue #89 (concurrency guard) prevents races but doesn't handle transient failures from tmux itself

## Solution

Add retry classification and policy to the domain layer:

```rust
impl TmuxError {
    /// Whether this error is transient and the operation can be retried
    pub fn is_retryable(&self) -> bool {
        matches!(self,
            TmuxError::TmuxNotRunning |  // Server may be restarting
            TmuxError::CommandFailed { .. } // Socket contention, etc.
        )
    }
}

pub struct RetryPolicy {
    pub max_attempts: u32,
    pub base_delay_ms: u64,
    pub max_delay_ms: u64,
    pub backoff_factor: f64,  // Exponential backoff multiplier
}

impl Default for RetryPolicy {
    fn default() -> Self {
        Self {
            max_attempts: 3,
            base_delay_ms: 50,
            max_delay_ms: 1000,
            backoff_factor: 2.0,
        }
    }
}

/// Execute a domain operation with retry policy
pub async fn with_retry<T, F, Fut>(
    policy: &RetryPolicy,
    operation: F,
) -> Result<T, TmuxError>
where
    F: Fn() -> Fut,
    Fut: std::future::Future<Output = Result<T, TmuxError>>,
{
    let mut last_error = None;
    for attempt in 0..policy.max_attempts {
        match operation().await {
            Ok(result) => return Ok(result),
            Err(e) if e.is_retryable() && attempt + 1 < policy.max_attempts => {
                let delay = /* calculate exponential backoff */;
                tokio::time::sleep(delay).await;
                last_error = Some(e);
            }
            Err(e) => return Err(e),
        }
    }
    Err(last_error.unwrap())
}
```

## Impact

- Domain-level resilience for transient tmux failures
- Clear classification of retryable vs permanent errors
- Configurable retry policy per deployment
- Reduces error rate visible to clients during tmux server instability
- Complements timeout (#87) and concurrency guard (#89)
