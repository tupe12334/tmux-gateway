# Add domain test builders and fixture factories for domain model types

As the domain model grows (TmuxSession, TmuxWindow, TmuxPane, TmuxServerInfo, and proposed types like WindowLayout, PaneContent, TmuxClient), testing becomes increasingly tedious. Every test must manually construct domain objects with all required fields, leading to verbose, brittle test setups.

## Problem

- Tests construct domain objects inline with all fields specified, even when only one field matters for the test
- No reusable test fixtures — each test file duplicates object construction
- Adding a field to a domain type breaks every test that constructs that type
- Integration tests in tests/*.rs construct expected objects manually for assertions
- No way to generate randomized domain objects for property-based testing
- As issues #82, #55, #73, #58 enrich domain models with more fields, test maintenance cost compounds

## Solution

Add a `testutil` module (behind `#[cfg(test)]` or a `test-support` feature flag) with builder pattern factories:

```rust
#[cfg(test)]
pub mod testutil {
    pub struct SessionBuilder {
        name: String,
        windows: u32,
        created: i64,
        attached: bool,
    }

    impl SessionBuilder {
        pub fn new() -> Self {
            Self {
                name: "test-session".to_string(),
                windows: 1,
                created: 1700000000,
                attached: false,
            }
        }

        pub fn name(mut self, name: &str) -> Self { self.name = name.to_string(); self }
        pub fn attached(mut self) -> Self { self.attached = true; self }
        pub fn windows(mut self, n: u32) -> Self { self.windows = n; self }
        pub fn build(self) -> TmuxSession { /* ... */ }
    }

    // Similar builders for TmuxWindow, TmuxPane, TmuxServerInfo
    // Plus helpers:
    pub fn sample_session() -> TmuxSession { SessionBuilder::new().build() }
    pub fn sample_window() -> TmuxWindow { WindowBuilder::new().build() }
}
```

## Scope

- New `testutil.rs` module in tmux-gateway-core (gated behind `#[cfg(test)]` or feature flag)
- Builders for all current domain types: TmuxSession, TmuxWindow, TmuxPane, TmuxServerInfo
- Convenience `sample_*()` functions for quick defaults
- Refactor existing unit tests to use builders where beneficial

## Impact

- Dramatically reduces test boilerplate across the project
- Adding fields to domain types only requires updating the builder default
- Enables property-based testing with randomized domain objects
- Accelerates implementation of all other domain issues
