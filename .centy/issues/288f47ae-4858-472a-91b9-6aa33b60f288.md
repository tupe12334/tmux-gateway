# Use typed timestamp for TmuxSession.created instead of locale-dependent String

## Problem

TmuxSession.created is a `String` parsed from tmux’s `#{session_created_string}` format variable. This format is locale-dependent and varies across systems, making it unreliable for comparisons, sorting, or programmatic use. The domain model should represent timestamps with a proper type, not a raw string.

## Current state

In sessions.rs, line 17:

````
.format("...#{session_created_string}...")
````

And the domain model stores it as:

````rust
pub struct TmuxSession {
    pub created: String,  // locale-dependent string like "Mon Jan 1 12:00:00 2024"
    ...
}
````

## Proposed change

1. Use tmux’s `#{session_created}` (unix epoch seconds) instead of `#{session_created_string}`
1. Change `TmuxSession.created` from `String` to `i64` (unix timestamp)
1. Each API layer can format the timestamp as needed for its transport:
   * REST: ISO 8601 string or unix timestamp in JSON
   * GraphQL: String field with ISO 8601 formatting
   * gRPC: google.protobuf.Timestamp or int64

This keeps the domain model precise and portable, letting transport layers decide presentation.

## Acceptance criteria

* [ ] TmuxSession.created is `i64` (unix epoch seconds)
* [ ] Parsing uses `#{session_created}` instead of `#{session_created_string}`
* [ ] Parse errors return a proper TmuxError instead of silent `unwrap_or(0)`
* [ ] All API layers format the timestamp appropriately for their transport
* [ ] Existing tests updated
