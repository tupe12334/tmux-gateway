# Add request logging and observability middleware

The server has basic tracing but no structured request-level logging or observability.

## Request Logging
- Add tower-http TraceLayer to Axum for request/response logging
- Log: method, path, status code, latency, request ID
- Add request ID middleware (generate UUID per request, propagate via headers)
- Include request ID in gRPC metadata

## Structured Logging
- Ensure all logs use structured fields (tracing spans/events)
- Add request context to all log entries within a request scope
- Configure JSON log format for production (RUST_LOG_FORMAT=json)

## Health Check Improvements
- REST /health should check tmux availability (not just return static response)
- gRPC health service should do the same
- Return degraded status if tmux is not running
