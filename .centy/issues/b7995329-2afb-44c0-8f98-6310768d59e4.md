# Add swap and move domain operations for windows and panes

## Problem

The domain currently supports creating, listing, killing, and renaming windows/panes, but lacks operations for rearranging them. tmux supports `swap-window`, `swap-pane`, and `move-window` which are fundamental for workspace organization and automation workflows.

These are core domain operations, not API concerns — they manipulate the tmux object graph (session→window→pane hierarchy) regardless of transport.

## Missing operations

1. **swap_window(source, destination)** — swap two windows within or across sessions
1. **move_window(source_target, destination_session)** — move a window from one session to another
1. **swap_pane(source, destination)** — swap two panes within a window

## Proposed changes

* Add `crates/tmux-gateway-core/src/swap_window.rs` with `swap_window(source: &str, dest: &str)`
* Add `crates/tmux-gateway-core/src/move_window.rs` with `move_window(source: &str, dest_session: &str)`
* Add `crates/tmux-gateway-core/src/swap_pane.rs` with `swap_pane(source: &str, dest: &str)`
* Add corresponding methods to the TmuxCommands trait
* Add validation for cross-session targets in the validation module
* All three APIs (REST, GraphQL, gRPC) will need handlers

## Acceptance criteria

* [ ] swap_window, move_window, swap_pane are available as core domain functions
* [ ] TmuxCommands trait includes the new methods
* [ ] Validation prevents invalid source/destination targets
* [ ] Unit tests for parsing and edge cases (swap with self, cross-session move)
* [ ] E2E tests covering the full operation lifecycle
