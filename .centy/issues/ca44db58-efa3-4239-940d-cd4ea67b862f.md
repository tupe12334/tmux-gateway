# Add optional feature flags for compile-time component selection

All three API protocols (REST, GraphQL, gRPC) are always compiled and enabled. Users who only need REST must still compile the GraphQL and gRPC stacks, increasing build time and binary size.

## What to do
- Define Cargo features: rest, graphql, grpc (all enabled by default)
- Gate each API module with cfg(feature = ...) 
- Gate corresponding dependencies with optional = true
- Add a "minimal" feature combining just rest
- Document feature flags in README

## Example
cargo build --no-default-features --features rest  # REST-only binary

## Acceptance criteria
- Each protocol can be independently disabled at compile time
- Default build includes all protocols (backward compatible)
- Binary size measurably decreases when protocols are excluded
