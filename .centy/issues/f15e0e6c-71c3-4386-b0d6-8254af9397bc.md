# Remove serde::Serialize from domain model types

## Problem
The domain types TmuxSession, TmuxWindow, and TmuxPane in tmux-gateway-core all derive serde::Serialize. This is an infrastructure/serialization concern leaking into the domain layer. The core crate should define pure domain models with no knowledge of how they will be serialized for transport.

## Current state
- sessions.rs: `#[derive(Debug, Clone, Serialize)] pub struct TmuxSession`
- list_windows.rs: `#[derive(Debug, Clone, Serialize)] pub struct TmuxWindow`
- list_panes.rs: `#[derive(Debug, Clone, Serialize)] pub struct TmuxPane`

The serde dependency exists in the core crate solely for these derives.

## Proposed change
1. Remove `Serialize` derive from all domain types in tmux-gateway-core
2. Remove the `serde` dependency from the core crate's Cargo.toml
3. Each API layer (REST, GraphQL, gRPC) already has its own response DTOs — they should map from domain types to their own serializable types (which they already do, e.g., SessionResponse in REST)

## Acceptance criteria
- [ ] TmuxSession, TmuxWindow, TmuxPane derive only Debug + Clone (no Serialize)
- [ ] serde is removed from tmux-gateway-core/Cargo.toml
- [ ] All API layers continue to work by mapping domain types to their own response types
- [ ] No public API behavior change
