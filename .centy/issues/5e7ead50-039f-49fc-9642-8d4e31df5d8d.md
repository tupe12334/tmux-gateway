# Expand tmux operations: list windows, list panes, send-keys, rename

The current API surface only covers 5 operations (ls, new-session, kill-session, kill-window, kill-pane). A production-ready tmux gateway needs more.

## New Operations to Add

1. **list_windows(session)** - List all windows in a session
1. **list_panes(window)** - List all panes in a window
1. **send_keys(target, keys)** - Send keystrokes to a pane
1. **rename_session(target, new_name)** - Rename a session
1. **rename_window(target, new_name)** - Rename a window
1. **new_window(session, name)** - Create a new window in a session
1. **split_window(target, horizontal/vertical)** - Split a window into panes
1. **capture_pane(target)** - Capture pane content as text

## Implementation Checklist

* Add functions to tmux-gateway-core
* Add methods to TmuxCommands trait (compiler enforces consistency)
* Implement in all 3 API layers (REST, GraphQL, gRPC)
* Update proto schema with new RPCs and messages
* Add tests for each new operation
* Update schemas/ exports
