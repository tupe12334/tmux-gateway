# Add GraphQL query depth and complexity limits

The GraphQL schema is built with Schema::build(QueryRoot, MutationRoot, EmptySubscription).finish() in src/api/graphql/schema.rs line 233-234 with no query depth or complexity limits configured.

## Problem:

* async-graphql supports .limit_depth() and .limit_complexity() on the schema builder
* Without limits, deeply nested or complex queries could consume excessive resources
* GraphQL introspection is also enabled by default with no option to disable in production

## Fix:

Add limits to the schema builder:

````rust
Schema::build(QueryRoot, MutationRoot, EmptySubscription)
    .limit_depth(10)
    .limit_complexity(100)
    .finish()
````

Consider making these configurable via env vars (GRAPHQL_MAX_DEPTH, GRAPHQL_MAX_COMPLEXITY) and allowing introspection to be disabled via GRAPHQL_INTROSPECTION=false.
