# Add domain-level session existence check

## Problem
There is no domain-level way to check if a session, window, or pane exists before performing operations on it. Currently, operations like kill_session or rename_session just forward to tmux and rely on tmux stderr parsing to detect "not found" — this is reactive error handling rather than proactive domain validation.

## Proposed change
Add existence-check functions to the domain core:

- `pub async fn session_exists(name: &str) -> Result<bool, TmuxError>`
- `pub async fn get_session(name: &str) -> Result<Option<TmuxSession>, TmuxError>`

These build on the existing list_sessions() and filter by name. They enable:
1. Pre-operation validation in domain logic (not just API layers)
2. Composable domain operations (e.g., "create session if not exists")
3. Better error messages with domain context rather than raw tmux stderr

## Acceptance criteria
- [ ] session_exists and get_session functions added to tmux-gateway-core
- [ ] Functions are API-agnostic — usable from any transport layer
- [ ] Unit tests cover: existing session, non-existing session, tmux not running
- [ ] Exported in lib.rs public API
