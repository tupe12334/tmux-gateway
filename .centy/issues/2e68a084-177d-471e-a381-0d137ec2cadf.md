# Add domain operations for tmux key binding management

tmux key bindings (\`bind-key\`, \`unbind-key\`, \`list-keys\`) control how users interact with tmux. The gateway has no domain operations for managing these bindings, preventing programmatic customization of tmux behavior.

## Problem
- No way to query current key bindings through the gateway
- No way to add or remove key bindings programmatically
- Customizing tmux behavior requires SSH access to run bind-key manually
- Key bindings are a core tmux configuration concept missing from the domain

## Proposed solution

\`\`\`rust
pub struct TmuxKeyBinding {
    pub key_table: String,     // e.g. "prefix", "root", "copy-mode"
    pub key: String,           // e.g. "C-b", "M-Up", "Space"
    pub command: String,       // bound tmux command
    pub note: Option<String>,  // optional -N note
}

/// List all key bindings, optionally filtered by key table.
pub async fn list_keys(table: Option<&str>) -> Result<Vec<TmuxKeyBinding>, TmuxError>

/// Bind a key to a tmux command.
pub async fn bind_key(
    table: Option<&str>,  // key table (None = prefix table)
    key: &str,
    command: &str,
) -> Result<(), TmuxError>

/// Remove a key binding.
pub async fn unbind_key(
    table: Option<&str>,
    key: &str,
) -> Result<(), TmuxError>
\`\`\`

Validate key names and commands for safety. This is a server-level operation — changes affect all sessions.

## Scope
- New \`key_bindings.rs\` module in \`tmux-gateway-core\`
- \`TmuxKeyBinding\` domain model
- Key name and command validation
- Add to \`TmuxCommands\` trait
- Complements #100 (source-file) for runtime tmux customization
