# Model pane cursor position and scrollback bounds in capture_pane output

capture_pane returns raw text content, but tmux exposes richer semantics: cursor position, scrollback buffer size, and whether output extends beyond the captured range. Callers cannot distinguish "full output" from "truncated output."

## Problem
- capture_pane returns a plain String with no metadata
- No indication of cursor row/column in the captured output
- No indication of total scrollback history size vs what was captured
- Monitoring and streaming use cases need to know if they missed output
- Issue #51 covers adding line range options, but not the metadata about what was captured

## What to do
Return a richer type from capture_pane:

pub struct CapturedPane {
    pub content: String,
    pub cursor_x: u32,
    pub cursor_y: u32,
    pub scroll_position: u32,
    pub history_size: u32,
}

Populate metadata by querying tmux format variables (#{cursor_x}, #{cursor_y}, #{scroll_position}, #{history_size}) alongside the capture.

## Acceptance criteria
- capture_pane returns content plus cursor/scroll metadata
- Callers can determine if more scrollback exists beyond what was captured
- API layers serialize the full CapturedPane structure
