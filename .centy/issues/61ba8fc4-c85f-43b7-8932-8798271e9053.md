# Add cross-session domain operations for multi-session management

Current domain operations work within a single session context. There are no operations for comparing, copying, or managing resources across sessions. Multi-session management is a common workflow for users managing multiple environments through the gateway.

## Problem
- No way to compare two sessions' structures (e.g. "does staging match production?")
- No way to copy a window from one session to another (link-window #88 shares, not copies)
- No way to list all panes across all sessions in a single query
- Operators managing multiple environments need cross-session visibility

## Proposed solution

\`\`\`rust
/// List all panes across all sessions, providing a global view.
pub async fn list_all_panes() -> Result<Vec<GlobalPane>, TmuxError>

pub struct GlobalPane {
    pub session: String,
    pub window_index: u32,
    pub window_name: String,
    pub pane_id: String,
    pub active: bool,
    pub current_command: String,
}

/// Compare the structure of two sessions (window/pane topology).
pub async fn compare_sessions(
    session_a: &str,
    session_b: &str,
) -> Result<SessionDiff, TmuxError>

pub struct SessionDiff {
    pub windows_only_in_a: Vec<String>,
    pub windows_only_in_b: Vec<String>,
    pub common_windows: Vec<WindowComparison>,
}

pub struct WindowComparison {
    pub name: String,
    pub pane_count_a: u32,
    pub pane_count_b: u32,
    pub layout_matches: bool,
}
\`\`\`

These are composite domain operations built on existing primitives (list_sessions, list_windows, list_panes).

## Scope
- New \`cross_session.rs\` module in \`tmux-gateway-core\`
- \`GlobalPane\`, \`SessionDiff\`, \`WindowComparison\` domain types
- Add to \`TmuxCommands\` trait
- Builds on existing list operations
