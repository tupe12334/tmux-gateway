# Enrich TmuxPane domain model with current path and command

## Problem
The TmuxPane domain model only captures id, width, height, and active status. Tmux can report additional useful information via format variables: `#{pane_current_path}` (the working directory) and `#{pane_current_command}` (the running process). This metadata is valuable for clients displaying pane overviews or deciding which pane to interact with.

## Current state
```rust
pub struct TmuxPane {
    pub id: String,
    pub width: u32,
    pub height: u32,
    pub active: bool,
}
```

Format string in list_panes.rs:
```
"#{pane_id}\t#{pane_width}\t#{pane_height}\t#{pane_active}"
```

## Proposed change
Add fields to TmuxPane:
```rust
pub struct TmuxPane {
    pub id: String,
    pub width: u32,
    pub height: u32,
    pub active: bool,
    pub current_path: String,     // working directory
    pub current_command: String,  // running process name
}
```

Update the format string to include `#{pane_current_path}` and `#{pane_current_command}`. Update the parsing logic (which should be a pure function per #36) to handle the additional fields.

## Acceptance criteria
- [ ] TmuxPane includes current_path and current_command
- [ ] Format string updated in list_panes
- [ ] Parsing handles the new fields
- [ ] All API layers expose the new fields in their DTOs
