# Add pipe_pane domain operation for streaming pane output to a command

Tmux pipe-pane feeds a pane's output to a shell command (e.g., logging to a file, streaming to a process). The gateway has capture_pane for snapshots but no way to set up continuous output piping.

## Problem
- capture_pane returns a point-in-time snapshot — no continuous output
- Issue #8 covers WebSocket streaming at the API layer, but the domain needs a mechanism to tap into pane output
- tmux pipe-pane -o -t target "cat >> /tmp/pane.log" is a core automation primitive
- No domain operation to start or stop output piping

## What to do
Add to tmux-gateway-core:

pub fn pipe_pane(target: &str, command: Option<String>) -> Result<(), TmuxError>

- When command is Some, starts piping output to the command
- When command is None, stops any existing pipe (tmux pipe-pane with no args)
- Validate target with validate_pane_target
- Validate command for shell injection (restrict to safe patterns or reject metacharacters)

## Acceptance criteria
- Pane output can be piped to a command via a domain operation
- Piping can be stopped
- Command validation prevents shell injection
- All API layers can expose start/stop pipe operations
