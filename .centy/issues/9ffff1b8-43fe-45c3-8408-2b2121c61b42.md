# Add show-messages domain operation for tmux server message log access

tmux \`show-messages\` provides access to the tmux server's internal message log — a record of recent events, errors, and debug information. This is essential for diagnosing tmux issues through the gateway without SSH access.

## Problem
- No way to access tmux's internal event/error log through the gateway
- When tmux commands fail with generic errors, the message log often has the real cause
- Operators diagnosing issues must SSH in and run \`show-messages\` manually
- Missing an important observability feature for production gateway deployments

## Proposed solution

\`\`\`rust
pub struct TmuxMessage {
    pub timestamp: i64,     // Unix timestamp
    pub message: String,    // message content
}

/// Retrieve the tmux server message log.
pub async fn show_messages() -> Result<Vec<TmuxMessage>, TmuxError>
\`\`\`

- Uses tmux \`show-messages\` command
- Parses the timestamp + message format
- No target validation needed (server-level operation)
- Returns messages in chronological order

## Scope
- New \`show_messages.rs\` module in \`tmux-gateway-core\`
- \`TmuxMessage\` domain model
- Add to \`TmuxCommands\` trait
- Complements #62 (domain tracing) with tmux-native logging
