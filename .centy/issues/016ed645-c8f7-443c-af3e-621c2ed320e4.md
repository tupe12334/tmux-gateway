# Add domain request context for cross-operation correlation and scoping

Multiple issues add cross-cutting concerns (auth #113, metrics #134, timeout #87, audit #127). Each needs request-scoped context. Add a RequestContext carrying correlation ID, principal, deadline, and metadata. All domain operations accept this context, enabling end-to-end tracing, authorization, and audit without parameter explosion.
