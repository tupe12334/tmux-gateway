# Stop re-exporting tmux_interface from core crate public API

## Problem
The core crate's lib.rs has `pub use tmux_interface;` which re-exports the entire tmux_interface crate as part of the domain's public API. This leaks an infrastructure/implementation detail through the domain boundary. Consumers of tmux-gateway-core should not need to know about tmux_interface — it's an implementation detail of how the domain talks to tmux.

## Current state
In crates/tmux-gateway-core/src/lib.rs line 35:
```rust
pub use tmux_interface;
```

This is used in:
- src/api/rest/routes.rs: `use crate::tmux::tmux_interface::Tmux;` for `check_tmux_available()`
- src/preflight.rs: `use tmux_gateway_core::tmux_interface::Tmux;` for version checking

Both of these usages should be replaced by domain-level functions (#44 addresses this with TmuxServerInfo).

## Proposed change
1. Remove `pub use tmux_interface;` from the core crate's public API
2. Keep tmux_interface as a private dependency within the core crate
3. Consumers that need tmux health/version info use domain functions instead

This should be done after or alongside #44 (TmuxServerInfo domain type).

## Acceptance criteria
- [ ] `pub use tmux_interface` removed from lib.rs
- [ ] No external code imports tmux_interface through the core crate
- [ ] tmux_interface remains available internally via `use tmux_interface` within core modules
- [ ] All external usages replaced with domain functions
