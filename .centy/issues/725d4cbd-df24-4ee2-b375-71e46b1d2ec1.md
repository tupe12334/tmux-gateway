# Add Display implementations for domain model types

## Problem

Domain types TmuxSession, TmuxWindow, and TmuxPane only derive Debug but don’t implement Display. Debug output is not user-friendly — it shows struct syntax like `TmuxSession { name: "foo", ... }`. When these types need to appear in logs, error messages, or CLI output, a human-readable Display format would be more appropriate.

## Proposed change

Add Display implementations:

````rust
impl fmt::Display for TmuxSession {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        write!(f, "{} ({} windows, {})", self.name, self.windows,
            if self.attached { "attached" } else { "detached" })
    }
}
````

Similar implementations for TmuxWindow and TmuxPane. These are pure domain formatting — not tied to any transport serialization.

## Acceptance criteria

* [ ] TmuxSession, TmuxWindow, TmuxPane implement Display
* [ ] Output is human-readable and concise
* [ ] Useful for logging and domain-level tracing (#62)
