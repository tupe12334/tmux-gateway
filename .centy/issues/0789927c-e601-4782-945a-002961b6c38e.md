# Add domain operations for tmux layout management

## Problem
Tmux supports automatic pane layout management via `select-layout` (even-horizontal, even-vertical, main-horizontal, main-vertical, tiled) and custom layout strings. The domain layer has no concept of layouts — clients that split windows into multiple panes have no way to arrange them automatically.

## Proposed change
Add to tmux-gateway-core:

```rust
pub enum PaneLayout {
    EvenHorizontal,
    EvenVertical,
    MainHorizontal,
    MainVertical,
    Tiled,
    Custom(String),
}

pub async fn select_layout(target: &str, layout: PaneLayout) -> Result<(), TmuxError>
```

This is a domain-level concept (how panes are arranged within a window), not a transport concern.

## Acceptance criteria
- [ ] PaneLayout enum defined in core crate
- [ ] select_layout function with target validation
- [ ] Added to TmuxCommands trait
- [ ] Available from all API layers
