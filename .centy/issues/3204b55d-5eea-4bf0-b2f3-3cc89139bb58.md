# Add break_pane and join_pane domain operations for pane-window restructuring

Tmux supports breaking a pane out into its own window (break-pane) and joining a pane from another window into the current one (join-pane). These structural operations are missing from the domain.

## Problem
- No way to promote a pane to its own window without recreating it
- No way to move a pane between windows without kill + recreate
- tmux break-pane -t pane creates a new window from the pane
- tmux join-pane -s source -t target moves a pane into a window
- These are common workflow operations for reorganizing tmux layouts

## What to do
Add to tmux-gateway-core:

pub fn break_pane(target: &str, window_name: Option<String>) -> Result<(), TmuxError>
pub fn join_pane(source: &str, target: &str, horizontal: bool) -> Result<(), TmuxError>

- break_pane validates target as pane target, optional window_name with validate_session_name
- join_pane validates both source and target as pane targets
- horizontal maps to -h flag (join side by side vs stacked)

## Acceptance criteria
- A pane can be broken out into its own window
- A pane can be moved from one window into another
- All API layers can expose these operations
