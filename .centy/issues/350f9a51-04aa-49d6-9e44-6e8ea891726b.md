# Add domain query combinators for filtering and searching domain collections

## Problem

API consumers frequently need to filter domain collections: "show only attached sessions", "find the active window", "list panes wider than 80 columns". Currently this filtering is pushed to each API layer independently, duplicating logic across REST, GraphQL, and gRPC.

The domain layer returns raw `Vec<TmuxSession>`, `Vec<TmuxWindow>`, `Vec<TmuxPane>` with no reusable query utilities.

## Proposed change

Add a `query` module in `tmux-gateway-core` with pure filtering and searching functions:

```rust
// Session queries
pub fn attached_sessions(sessions: &[TmuxSession]) -> Vec<&TmuxSession>;
pub fn detached_sessions(sessions: &[TmuxSession]) -> Vec<&TmuxSession>;
pub fn find_session_by_name<'a>(sessions: &'a [TmuxSession], name: &str) -> Option<&'a TmuxSession>;
pub fn sessions_created_after(sessions: &[TmuxSession], timestamp: i64) -> Vec<&TmuxSession>;
pub fn sort_sessions_by_created(sessions: &mut [TmuxSession], ascending: bool);

// Window queries
pub fn active_window(windows: &[TmuxWindow]) -> Option<&TmuxWindow>;
pub fn find_window_by_name<'a>(windows: &'a [TmuxWindow], name: &str) -> Option<&'a TmuxWindow>;
pub fn find_window_by_index(windows: &[TmuxWindow], index: u32) -> Option<&TmuxWindow>;
pub fn windows_with_multiple_panes(windows: &[TmuxWindow]) -> Vec<&TmuxWindow>;

// Pane queries
pub fn active_pane(panes: &[TmuxPane]) -> Option<&TmuxPane>;
pub fn panes_wider_than(panes: &[TmuxPane], min_width: u32) -> Vec<&TmuxPane>;
pub fn panes_taller_than(panes: &[TmuxPane], min_height: u32) -> Vec<&TmuxPane>;
pub fn largest_pane(panes: &[TmuxPane]) -> Option<&TmuxPane>;
```

All functions are pure — they operate on already-fetched data with zero I/O.

## Why this matters

- Eliminates duplicated filtering logic across REST, GraphQL, and gRPC layers
- Pure functions are trivially testable with arbitrary test data
- Foundation for future features: GraphQL field-level filtering, REST query parameters
- Complements issue #86 (find-window for text search) with structural queries

## Acceptance criteria

- [ ] `query` module in `tmux-gateway-core` with functions for sessions, windows, and panes
- [ ] All functions are pure (no I/O, no tmux dependency)
- [ ] Unit tests with various collection sizes (empty, single, many)
- [ ] Tests for edge cases: no active window, no matching name, all attached
- [ ] At least 3 functions used by an API layer to replace inline filtering
