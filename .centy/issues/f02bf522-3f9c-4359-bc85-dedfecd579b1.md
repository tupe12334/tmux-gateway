# Replace String errors with custom error types

All core functions return `Result<_, String>`. This loses structured error information and makes error handling in API layers brittle (string matching).

## Proposed Changes
- Create a `TmuxError` enum in tmux-gateway-core:
  - `SessionNotFound(String)`
  - `WindowNotFound(String)`
  - `PaneNotFound(String)`
  - `TmuxNotRunning`
  - `CommandFailed { command: String, stderr: String }`
  - `InvalidTarget(String)`
- Implement `std::error::Error` and `Display`
- Map to appropriate HTTP status codes (404, 500, 400)
- Map to appropriate gRPC status codes (NOT_FOUND, INTERNAL, INVALID_ARGUMENT)
- Use `thiserror` crate for derive macros

## Benefits
- Proper HTTP/gRPC status codes per error type
- Structured error responses for clients
- No string parsing for error classification
