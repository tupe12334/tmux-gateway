# package.json license field says ISC but project uses MIT

## Problem

The package.json file (used for husky git hooks) has `"license": "ISC"` but the Rust project is licensed under MIT (Cargo.toml `license = "MIT"`). This is a metadata inconsistency that could cause confusion about the project's actual license.

## Current state

* Cargo.toml: `license = "MIT"`
* package.json: `"license": "ISC"`

## Proposed change

Change package.json `"license"` field to `"MIT"` to match the actual project license.

## Acceptance criteria

* [ ] package.json license changed from "ISC" to "MIT"
