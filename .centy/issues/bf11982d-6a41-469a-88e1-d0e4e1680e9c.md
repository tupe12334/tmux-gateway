# Add domain batch operation support for atomic multi-step workflows

Individual domain operations work well for simple requests, but real-world tmux management often requires multiple operations that should succeed or fail together. There is no domain concept for batching or transactional execution.

## Problem
- Creating a development workspace (new session + multiple windows + pane splits + send initial commands) requires many sequential API calls
- If step 3 of 5 fails, the caller is left with a partially-created workspace and no automatic cleanup
- Issue #46 proposes composite operations for specific workflows, but a general batch mechanism is more flexible
- Each API call pays network overhead; batching reduces round trips

## Proposed solution
Add a domain-level batch operation executor:

\`\`\`rust
pub enum BatchOperation {
    NewSession { name: String },
    NewWindow { session: String, name: String },
    SplitWindow { target: String, horizontal: bool },
    SendKeys { target: String, keys: Vec<String> },
    RenameSession { target: String, new_name: String },
    RenameWindow { target: String, new_name: String },
    KillSession { target: String },
    KillWindow { target: String },
    KillPane { target: String },
}

pub struct BatchResult {
    pub results: Vec<Result<(), TmuxError>>,
    pub completed: usize,
    pub failed: usize,
}

/// Execute a sequence of operations. Stops at first failure if \`stop_on_error\` is true.
pub async fn execute_batch(
    operations: Vec<BatchOperation>,
    stop_on_error: bool,
) -> BatchResult
\`\`\`

Design considerations:
- Each operation in the batch is validated before execution begins
- \`stop_on_error: true\` provides fail-fast semantics
- \`stop_on_error: false\` executes all operations, collecting failures
- No automatic rollback (tmux operations are not truly transactional), but partial failure is clearly reported
- API layers serialize/deserialize the batch enum as appropriate for their protocol

## Scope
- New \`batch.rs\` module in \`tmux-gateway-core\`
- \`BatchOperation\` enum, \`BatchResult\` struct
- \`execute_batch\` function
- Add batch execution to \`TmuxCommands\` trait
- Supersedes #46 (composite operations) as a more general solution
