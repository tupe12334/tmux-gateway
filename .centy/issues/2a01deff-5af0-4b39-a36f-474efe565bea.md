# Add schema breaking change detection to CI

The CI has schema-drift detection (checks if generated schemas match committed ones), but it does not detect breaking changes in the API schemas between PRs. A client relying on a removed field or renamed endpoint would break silently.

## What to do
- Add a CI step that compares the PR's OpenAPI spec against main's spec for breaking changes
- Use oasdiff or openapi-diff to detect: removed endpoints, removed fields, changed types, narrowed enums
- For gRPC: use buf breaking to detect proto breaking changes
- Fail CI on breaking changes unless explicitly acknowledged

## Acceptance criteria
- PRs that remove or rename REST endpoints fail CI with a clear message
- PRs that remove proto fields or change field types fail CI
- A mechanism exists to acknowledge intentional breaking changes (e.g., a label or changelog entry)
