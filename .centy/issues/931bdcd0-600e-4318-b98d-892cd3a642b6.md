# Add new_window options for start directory and initial command

new_window only accepts a session name and window name. Like new_session (#79), tmux new-window supports a start directory (-c) and initial shell command. These are essential for automation.

## Problem
- new_window(session, name) creates a window with default shell in the session's directory
- No way to specify a working directory for the new window
- No way to run an initial command (e.g., new-window -n logs "tail -f /var/log/syslog")
- Consistent with #79 — both create operations should accept the same options pattern

## What to do
Extend new_window to accept options:

pub struct NewWindowOptions {
    pub session: String,
    pub name: String,
    pub start_directory: Option<String>,
    pub shell_command: Option<String>,
}

pub fn new_window(options: NewWindowOptions) -> Result<String, TmuxError>

- Validate start_directory for path traversal
- Validate shell_command for injection
- Map to tmux new-window -d -t session -n name [-c dir] [command]

## Acceptance criteria
- Windows can be created with a specific working directory and initial command
- Consistent options pattern with new_session (#79)
- All API layers can expose these options
