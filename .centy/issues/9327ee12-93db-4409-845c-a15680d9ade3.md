# Add pane content search with regex support for captured output

Issue #86 covers `find-window` for searching across panes by name/title, and #120 covers copy-mode search (interactive). But there is no domain operation for searching within a single pane's captured content using regex patterns. This is a core use case for CI pipelines, monitoring scripts, and automated testing that need to verify specific output appeared in a pane.

## Problem

- `capture_pane` returns raw text — clients must implement their own search logic
- No domain concept of content matching or pattern extraction
- CI/test automation needs "wait until output contains X" which requires repeated capture + client-side search
- `find-window` (#86) searches window/pane names, not content
- Copy-mode search (#120) is interactive, not programmatic
- No way to extract matched content or line numbers from pane output

## Solution

Add domain operations for pane content search:

```rust
pub struct ContentMatch {
    pub line_number: u32,        // 0-indexed line in captured content
    pub line: String,            // Full matching line
    pub match_start: u32,        // Byte offset of match within line
    pub match_end: u32,          // End of match
    pub matched_text: String,    // The matched substring
}

pub struct SearchOptions {
    pub pattern: String,         // Regex pattern
    pub case_insensitive: bool,
    pub max_matches: Option<u32>,
    pub include_history: bool,   // Search scrollback (from #51)
}

/// Search pane content for a regex pattern
```

## Impact

- Enables "assert pane contains X" for automated testing
- Reduces client-side complexity for output monitoring
- Structured match results enable precise content extraction
- Foundation for "wait until output matches" polling patterns
- Complements capture_pane (#51), find-window (#86), and change detection (#115)
