# Add input validation to new tmux operations (send_keys, split_window, rename, new_window, list)

Issue #3 (input validation) was closed before 8 new operations were added. These new operations lack the validation pattern established in kill_session/kill_pane/kill_window/capture_pane.

## Missing validation:

* send_keys(): no validate_pane_target(target) call
* split_window(): no validate_pane_target(target) call
* rename_session(): no validate_session_target(target), no new_name validation
* rename_window(): no validate_window_target(target) call
* new_window(): no validate_session_target(session), no name validation
* list_panes(): no validate_window_target(target) call
* list_windows(): no validate_session_target(target) call

## Also missing:

* validate_window_name() function does not exist yet (only validate_session_name exists)
* send_keys() should reject empty keys array
