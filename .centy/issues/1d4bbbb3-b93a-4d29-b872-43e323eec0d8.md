# Enrich OpenAPI spec with field descriptions and examples

The OpenAPI spec lacks per-field descriptions and request/response examples, making the Swagger UI less useful for API consumers.

## Missing Documentation
- No field descriptions on SessionResponse, NewSessionRequest, KillSessionRequest etc.
- No request/response examples in OpenAPI spec
- No error response schemas documented (currently just returns String on error)
- No description of session name format constraints

## Implementation
- Add #[schema(example = ...)] attributes to all utoipa structs
- Add field-level documentation via #[schema(description = ...)]
- Define a structured ErrorResponse type (code, message, request_id)
- Document all possible error status codes per endpoint
- Add example values that work against a real tmux server
