# Replace .unwrap() calls in server startup with proper error handling

src/main.rs has multiple .unwrap() calls during server initialization that will cause unhandled panics with no useful error messages.

## Problem locations (src/main.rs):
- Line ~75: TcpListener::bind(&http_addr).await.unwrap() - panics if HTTP port in use
- Line ~83: axum::serve().await.unwrap() - panics on serving errors
- Line ~88: grpc_addr.parse().unwrap() - panics on invalid address format
- Line ~92: build_v1().unwrap() - panics on reflection service build failure
- Line ~107: serve_with_shutdown().await.unwrap() - panics on gRPC serving errors

## Solution:
- Replace .unwrap() with proper error handling using anyhow or custom errors
- Log clear error messages before exiting (e.g. 'HTTP port 8080 already in use')
- Ensure cleanup happens on startup failure
- Consider adding these checks to src/preflight.rs where appropriate
