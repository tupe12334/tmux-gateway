# Add buffer list and management domain operations

## Problem

Tmux maintains a buffer stack (`list-buffers`, `show-buffer`, `set-buffer`, `delete-buffer`, `load-buffer`, `save-buffer`) that stores copied text. Issue #79 covers paste buffer operations (paste-buffer), but there are no domain operations for listing, reading, creating, or deleting individual named buffers.

Buffers are the clipboard system of tmux — essential for workflows involving text extraction from panes, inter-pane data transfer, and programmatic content injection.

## Proposed change

Add buffer management domain operations:

```rust
/// A tmux buffer entry.
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct TmuxBuffer {
    pub name: String,
    pub size: usize,
    pub created: i64,
}

/// List all buffers in the buffer stack.
pub async fn list_buffers(executor: &impl TmuxExecutor) -> Result<Vec<TmuxBuffer>, TmuxError>;

/// Get the content of a specific buffer.
pub async fn get_buffer(
    executor: &impl TmuxExecutor,
    name: &str,
) -> Result<String, TmuxError>;

/// Create or overwrite a named buffer with the given content.
pub async fn set_buffer(
    executor: &impl TmuxExecutor,
    name: &str,
    content: &str,
) -> Result<(), TmuxError>;

/// Delete a named buffer.
pub async fn delete_buffer(
    executor: &impl TmuxExecutor,
    name: &str,
) -> Result<(), TmuxError>;
```

The `TmuxBuffer` value object provides metadata; `get_buffer` retrieves content separately (buffers can be large).

## Why this matters

- Completes the buffer subsystem: #79 covers paste-buffer, this covers CRUD
- Enables programmatic clipboard workflows: capture pane content → store in buffer → retrieve from API
- `TmuxBuffer` is a clean value object; operations are pure domain functions
- Essential for automation: extract command output, transfer between sessions, inject content
- API-agnostic: all three APIs benefit from the same domain operations

## Acceptance criteria

- [ ] `TmuxBuffer` value object with name, size, created timestamp
- [ ] `list_buffers` parses `tmux list-buffers` output
- [ ] `get_buffer(name)` retrieves buffer content via `tmux show-buffer -b {name}`
- [ ] `set_buffer(name, content)` creates/overwrites via `tmux set-buffer -b {name}`
- [ ] `delete_buffer(name)` removes via `tmux delete-buffer -b {name}`
- [ ] Buffer name validation (no shell metacharacters)
- [ ] Unit tests with mock executor for each operation
- [ ] Parse error handling for malformed buffer list output
