# Add resize_pane domain operation with minimum size validation

The domain exposes pane width/height via list_panes but provides no operation to resize panes. Tmux has resize-pane with constraints (minimum 1 row/col), but the gateway offers no way to invoke or validate resizes.

## Problem
- TmuxPane exposes width and height as read-only
- No domain function to resize panes
- No validation of minimum pane dimensions (tmux enforces 1x1 minimum)
- Clients must work around this by using send_keys with tmux hotkeys

## What to do
Add to tmux-gateway-core:

pub async fn resize_pane(target: &str, width: Option<u32>, height: Option<u32>) -> Result<(), TmuxError>

- Validate target with validate_pane_target
- Validate dimensions are > 0 if provided
- Map to tmux resize-pane -x/-y flags
- Return appropriate error if resize exceeds available space

## Acceptance criteria
- Panes can be resized via a domain operation
- Invalid dimensions (0, negative) are rejected at the domain level before reaching tmux
- All three API layers can expose this operation
