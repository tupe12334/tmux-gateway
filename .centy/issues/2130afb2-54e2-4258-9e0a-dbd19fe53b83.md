# Add rustfmt.toml and clippy.toml for consistent code style enforcement

No rustfmt.toml or clippy.toml exists. Formatting and lint rules are implicit, which can lead to inconsistencies across contributors and CI surprises.

## What to do
- Add rustfmt.toml with explicit settings (edition, max_width, imports_granularity, etc.)
- Add clippy.toml or .clippy.toml with project-specific lint configuration
- Ensure CI clippy and fmt checks use these configs
- Document any non-default style choices

## Acceptance criteria
- cargo fmt produces identical output for all contributors
- Clippy rules are explicitly configured rather than relying on defaults
