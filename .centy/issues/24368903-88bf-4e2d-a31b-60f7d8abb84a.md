# Document recursion_limit requirement in lib.rs

## Problem
The main crate's lib.rs has `#![recursion_limit = "1024"]` without any comment explaining why. This was likely needed for the gRPC macro expansion in `src/api/grpc/server.rs` and `src/api/grpc/messages.rs`, but without documentation a contributor might try to remove it and face cryptic compilation errors.

## Current state
```rust
#![recursion_limit = "1024"]

pub mod api;
pub mod export_schemas;
// ...
```

## Proposed change
Add a brief comment:
```rust
// Required for the gRPC macro expansions in api::grpc (grpc_service! and proto_messages!)
#![recursion_limit = "1024"]
```

Also investigate if the limit can be reduced or if the macros can be restructured to avoid deep recursion.

## Acceptance criteria
- [ ] Comment explaining why the recursion limit is needed
- [ ] Investigation of whether the limit can be reduced
