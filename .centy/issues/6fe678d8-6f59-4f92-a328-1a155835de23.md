# Dockerfile port defaults mismatch with .env.example

The Dockerfile sets different default ports than .env.example, causing confusion.

## Dockerfile (lines 24-27):

````
ENV HTTP_PORT=3020
ENV GRPC_PORT=50251
EXPOSE 3020 50251
````

## .env.example:

````
HTTP_PORT=8080
GRPC_PORT=50051
````

## Impact:

* Docker users get different ports than local dev users
* README examples reference 8080/50051 which won’t work with Docker defaults
* EXPOSE doesn’t match .env.example

## Fix:

* Align Dockerfile ENV defaults with .env.example (8080 / 50051)
* Or document the discrepancy clearly
* Update EXPOSE to match
