# Add domain operations for session and pane environment variables

Tmux sessions and panes carry environment variables that affect spawned processes, but the gateway has no operations to query or set them. This blocks automation workflows that need to configure subprocess environments.

## Problem
- No way to read a session's environment (tmux show-environment)
- No way to set environment variables on a session (tmux set-environment)
- Environment inheritance (session -> window -> pane) is an unmodeled domain concept
- Automation that spawns processes in panes cannot pre-configure the environment through the gateway

## What to do
Add to tmux-gateway-core:

pub async fn get_environment(session: &str) -> Result<Vec<EnvVar>, TmuxError>
pub async fn set_environment(session: &str, name: &str, value: &str) -> Result<(), TmuxError>
pub async fn unset_environment(session: &str, name: &str) -> Result<(), TmuxError>

pub struct EnvVar { pub name: String, pub value: String }

- Validate session name, env var name (alphanumeric + underscore), and value (no null bytes)
- Map to tmux show-environment and set-environment commands

## Acceptance criteria
- Environment variables can be read and written through domain operations
- Env var names are validated (no shell injection via env names)
- All three API layers can expose these operations
