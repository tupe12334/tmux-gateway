# Make schema export optional in production via env var

export\_schemas::export\_all() runs on every server startup (src/main.rs line 24), writing 4 files to the schemas/ directory. In production this is unnecessary disk I/O and could fail if the directory isn’t writable.

## Problem:

* Schema files are only useful for development and CI drift detection

* Production containers may have read-only filesystems

* Adds startup latency (proto compilation + file writes)

* Uses .expect() which will panic if writes fail (related to #23)

## Fix:

* Gate schema export behind an env var: EXPORT\_SCHEMAS=true (default true)

* Or only export when schemas/ directory exists and is writable

* Consider moving schema export to a separate binary/subcommand (already exists as src/bin/export\_schemas.rs)
