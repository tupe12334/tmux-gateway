# Add pane synchronization domain operation (synchronize-panes)

Tmux supports synchronize-panes mode where input is sent to all panes in a window simultaneously. This is a key automation feature for managing multiple similar processes (e.g., running the same command across multiple servers).

## Problem
- send_keys targets a single pane — no way to broadcast input to all panes in a window
- The only workaround is calling send_keys once per pane, which is non-atomic and slow
- tmux set-window-option synchronize-panes on/off toggles this mode
- This is a common tmux workflow not modeled in the domain

## What to do
Add to tmux-gateway-core:

pub fn set_pane_synchronization(target: &str, enabled: bool) -> Result<(), TmuxError>
pub fn get_pane_synchronization(target: &str) -> Result<bool, TmuxError>

- target is a window target
- Maps to tmux set-window-option -t <target> synchronize-panes on/off
- Query via tmux show-window-options -v synchronize-panes

## Acceptance criteria
- Pane synchronization can be toggled via domain operations
- Current synchronization state can be queried
- All three API layers can expose these operations
