# Introduce domain value objects for session, window, and pane targets

Session names, window targets, and pane targets are passed as raw &str throughout the domain layer. This allows invalid values to flow through the system and scatters validation logic across call sites.

## Problem
- new_session("foo") and kill_session("foo") both take &str but have different validation rules
- Validation is procedural (validate_session_name(), validate_session_target()) rather than type-level
- A pane target string could accidentally be passed where a session name is expected — no compiler protection

## What to do
Create validated newtype wrappers in tmux-gateway-core:
- SessionName — alphanumeric + -_. , max 256 chars, validated on construction
- SessionTarget — valid session name or pattern
- WindowTarget — format: session:window
- PaneTarget — format: session:window.pane

Each type validates in its constructor and returns Result<Self, ValidationError>. Domain functions accept these types instead of &str:
- new_session(name: &SessionName)
- kill_session(target: &SessionTarget)
- kill_window(target: &WindowTarget)
- kill_pane(target: &PaneTarget)

## Acceptance criteria
- Passing an invalid string fails at construction time, not at tmux execution time
- Compiler prevents passing a PaneTarget where a SessionName is expected
- Existing validation logic moves into constructors (no duplication)
- All API layers construct value objects from user input before calling domain functions
