# Log effective configuration at startup

The server doesn’t log its effective configuration at startup, making it hard to diagnose configuration issues in production.

## What to log at info level:

* HTTP listen address and port
* gRPC listen address and port
* Effective CORS origins (after parsing)
* Shutdown timeout value
* tmux binary version detected during preflight
* Number of valid vs invalid CORS origins parsed

## Why:

* Users can’t verify that their environment variables took effect
* CORS misconfiguration is invisible (see issue #24)
* Debugging requires restarting with RUST_LOG=debug and reading through noise

## Files to modify:

* src/main.rs (add info! logs after config parsing, before server start)
