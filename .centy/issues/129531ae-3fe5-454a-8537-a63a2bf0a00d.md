# Add resize_pane domain operation

## Problem

Tmux supports resizing panes with `resize-pane -U/-D/-L/-R/-x/-y` but the domain layer has no operation for it. For clients controlling tmux remotely (the core use case of tmux-gateway), resizing panes is a fundamental capability that’s currently missing.

## Proposed change

Add to tmux-gateway-core:

````rust
pub enum ResizeDirection {
    Up(u32),
    Down(u32),
    Left(u32),
    Right(u32),
}

pub async fn resize_pane(target: &str, direction: ResizeDirection) -> Result<(), TmuxError>
````

Alternatively, support absolute sizing:

````rust
pub async fn resize_pane_absolute(target: &str, width: Option<u32>, height: Option<u32>) -> Result<(), TmuxError>
````

* [ ] ResizeDirection enum defined in core crate
* [ ] resize_pane function with target validation
