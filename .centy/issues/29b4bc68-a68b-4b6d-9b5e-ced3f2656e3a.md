# Add PartialEq and Eq derives to domain model types

## Problem
The domain types TmuxSession, TmuxWindow, and TmuxPane only derive Debug and Clone. Without PartialEq/Eq, these types cannot be compared in assertions, used as hash map keys, or checked for equality in domain logic. This limits testability and composability.

## Current state
```rust
#[derive(Debug, Clone, Serialize)]  // no PartialEq
pub struct TmuxSession { ... }
```

Tests that need to verify domain results must manually compare individual fields rather than asserting on the whole struct.

## Proposed change
Add `PartialEq, Eq` derives to TmuxSession, TmuxWindow, TmuxPane, TmuxError, and ValidationError (ValidationError already has PartialEq + Eq).

This is a non-breaking additive change that enables:
- `assert_eq!(expected_session, actual_session)` in tests
- Domain-level deduplication and comparison logic
- Use in collections that require Eq (HashSet, HashMap keys)

## Acceptance criteria
- [ ] TmuxSession, TmuxWindow, TmuxPane derive PartialEq + Eq
- [ ] TmuxError derives PartialEq + Eq
- [ ] No behavior change in existing code
- [ ] Existing tests still pass
