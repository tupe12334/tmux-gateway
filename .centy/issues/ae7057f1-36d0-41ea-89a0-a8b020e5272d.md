# Run cargo deny in CI for dependency security scanning

A deny.toml config exists with license and advisory rules, but cargo deny is only run via .husky pre-commit hooks (easily bypassed). It should run in CI to catch security advisories and license violations.

## What to do
- Add a cargo-deny job to .github/workflows/ci.yml
- Run: cargo deny check licenses advisories sources
- Fail the build on policy violations
- Review and address the currently-ignored RUSTSEC-2024-0436 advisory

## Acceptance criteria
- CI fails if a dependency has a known vulnerability or disallowed license
- The ignored advisory has a tracking comment explaining the plan to resolve it
