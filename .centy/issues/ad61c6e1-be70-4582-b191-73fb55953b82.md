# Replace stringly-typed stderr parsing with structured tmux error detection

TmuxError::from_stderr() classifies errors by searching stderr for substrings like "no server running", "session not found", etc. This is brittle — tmux error messages can change across versions or locales.

## Problem
- from_stderr() uses stderr_lower.contains("no server running") to detect TmuxNotRunning
- "session not found" detection is a substring match that could false-positive on similar messages
- No tmux version awareness — error messages differ between tmux 3.x and 4.x
- If tmux adds a new error message pattern, it falls through to generic CommandFailed

## What to do
- Document which tmux versions are supported and their known error message formats
- Use exit codes in combination with stderr patterns (tmux uses specific exit codes for some failures)
- Consider matching against tmux exit status first, stderr patterns second
- Add integration tests that verify error classification against the installed tmux version
- Extract error patterns into named constants for maintainability

## Acceptance criteria
- Error classification uses exit codes where available, not just string matching
- Known error patterns are documented with the tmux versions they apply to
- Tests verify correct error classification for common failure scenarios
