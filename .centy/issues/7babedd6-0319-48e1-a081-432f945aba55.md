# Pre-push hook runs cargo fmt twice redundantly

## Problem
The pre-push hook in .husky/pre-push runs `cargo fmt` twice:
```
cargo fmt --all -- --check    # line 1: check for formatting issues
cargo fmt --all               # line 2: actually format (modifies files)
```

This is contradictory — line 1 checks if formatting is correct (fails if not), and line 2 applies formatting regardless. If line 1 fails, the push is rejected before line 2 runs. If line 1 passes, line 2 is a no-op.

The second `cargo fmt --all` (without --check) should not be in a pre-push hook because it silently modifies working tree files during a push, which is unexpected behavior.

## Proposed change
Remove the redundant `cargo fmt --all` (line 2). Keep only the `--check` variant that verifies formatting without modifying files. Developers should run `cargo fmt` manually before pushing.

## Acceptance criteria
- [ ] Pre-push hook only has `cargo fmt --all -- --check`
- [ ] No silent file modifications during push
