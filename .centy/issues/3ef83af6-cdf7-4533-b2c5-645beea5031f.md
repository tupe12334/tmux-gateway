# Split REST routes.rs into separate handler and type modules

src/api/rest/routes.rs is over 500 lines, mixing request/response type definitions, handler functions, OpenAPI annotations, and router construction in a single file. This hurts readability and makes code review harder as endpoints grow.

## What to do
- Split into:
  - src/api/rest/types.rs — Request/response structs with serde and utoipa derives
  - src/api/rest/handlers.rs — Handler functions
  - src/api/rest/mod.rs — Router construction and ApiDoc
- Keep the public API (router(), ApiDoc) unchanged

## Acceptance criteria
- No file in src/api/rest/ exceeds ~200 lines
- All existing tests continue to pass without modification
- Public API is unchanged
