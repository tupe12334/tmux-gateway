# capture_pane REST endpoint reuses KillTargetRequest instead of own type

In src/api/rest/routes.rs line 490, the capture_pane handler uses KillTargetRequest as its query parameter type:

```rust
async fn capture_pane(
    axum::extract::Query(params): axum::extract::Query<KillTargetRequest>,
)
```

## Problem:
- KillTargetRequest is semantically wrong for a read operation
- OpenAPI/Swagger docs show 'KillTargetRequest' as the schema name for capture_pane parameters
- Confusing for API consumers reading the generated OpenAPI spec

## Fix:
- Create a dedicated CapturePaneRequest (or generic TargetQuery) struct
- Use it in the capture_pane handler
- This also affects the OpenAPI schema output in schemas/openapi.json
