# Fix silent parsing failures in list operations (unwrap_or(0))

list_panes, list_windows, and list_sessions parse numeric fields from tmux output using .parse().unwrap_or(0). If tmux returns unexpected data, clients silently receive 0 instead of an error.

## Affected files:
- crates/tmux-gateway-core/src/list_panes.rs (width, height at lines 45-46)
- crates/tmux-gateway-core/src/list_windows.rs (index, panes at lines 44, 46)
- crates/tmux-gateway-core/src/sessions.rs (windows at line 45)

## Problem:
```rust
width: parts[1].parse().unwrap_or(0),  // Silent failure
height: parts[2].parse().unwrap_or(0), // Silent failure
```

## Solution:
- Return TmuxError when parsing fails instead of defaulting to 0
- Or log a warning and skip the malformed entry
- Consider a dedicated TmuxError variant for parse failures
