# Add domain operation result envelope with execution metadata

Every domain operation returns its own ad-hoc type (String, Vec<TmuxSession>, (), etc.) with no consistent metadata. For production observability, clients need execution context: how long the operation took, which tmux server handled it, and correlation information. This is distinct from tracing spans (#62) which are internal to the server.

## Problem

- No way for clients to know how long a domain operation took
- No correlation between a request and the tmux command that executed
- Different operations return completely different result shapes with no common envelope
- Debugging production issues requires correlating API logs with tmux behavior
- Rate limiting (#16) and timeout (#87) decisions need operation timing data

## Solution

Add a generic result envelope that wraps all domain operation results:

```rust
pub struct OperationResult<T> {
    pub data: T,
    pub metadata: OperationMetadata,
}

pub struct OperationMetadata {
    pub operation: &'static str,       // e.g., "list_sessions", "send_keys"
    pub duration_ms: u64,              // Wall clock execution time
    pub tmux_command: String,          // The tmux command that was executed
    pub target: Option<String>,        // The target (session/window/pane)
}

// Usage in domain operations:
pub async fn list_sessions() -> Result<OperationResult<Vec<TmuxSession>>, TmuxError> {
    let start = Instant::now();
    let sessions = /* execute tmux command */;
    Ok(OperationResult {
        data: sessions,
        metadata: OperationMetadata {
            operation: "list_sessions",
            duration_ms: start.elapsed().as_millis() as u64,
            tmux_command: "list-sessions".to_string(),
            target: None,
        },
    })
}
```

## Impact

- Consistent metadata across all domain operations
- API layers can expose timing and correlation data to clients
- Enables client-side performance monitoring and debugging
- Foundation for operation auditing and logging
- Complements tracing spans (#62) with client-visible metadata
