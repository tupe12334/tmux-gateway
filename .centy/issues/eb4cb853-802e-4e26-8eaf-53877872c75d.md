# Add TmuxClient domain model and list-clients domain operation

The domain model currently has no representation for tmux clients (terminal connections attached to sessions). The `list-clients` tmux command provides essential visibility into who/what is connected to the tmux server.

## Problem
- No way to query which terminals are attached to sessions
- Cannot determine if a session is actively being used vs. idle
- Missing fundamental tmux entity from the domain model

## Proposed solution
Add a `TmuxClient` domain model type:

```rust
pub struct TmuxClient {
    pub tty: String,           // e.g. "/dev/ttys003"
    pub session: String,       // attached session name
    pub width: u32,
    pub height: u32,
    pub created: String,       // or typed timestamp per #42
    pub activity: String,      // last activity timestamp
}
```

Add a `list_clients` domain operation:
- `list_clients() -> Result<Vec<TmuxClient>, TmuxError>` — list all connected clients
- `list_clients_for_session(target: &str) -> Result<Vec<TmuxClient>, TmuxError>` — clients attached to a specific session

## Scope
- New domain model type in `tmux-gateway-core`
- New domain operation module `list_clients.rs`
- Add to `TmuxCommands` trait so all API layers implement it
- Parse tmux `list-clients` format string output into `TmuxClient` structs
