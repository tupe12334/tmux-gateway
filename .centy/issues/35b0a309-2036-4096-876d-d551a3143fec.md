# Add window layout management domain operations (select-layout, list presets)

Tmux windows have layout arrangements (even-horizontal, even-vertical, main-horizontal, main-vertical, tiled) that control how panes are positioned. The gateway provides no way to query or change layouts, forcing users to manually resize panes.

## Problem
- split_window creates panes but the resulting layout cannot be controlled
- No domain concept for layout presets
- Complex pane arrangements require manual resize_pane calls per pane
- tmux select-layout applies a preset layout in one operation — much simpler

## What to do
Add to tmux-gateway-core:

pub enum LayoutPreset {
    EvenHorizontal,
    EvenVertical,
    MainHorizontal,
    MainVertical,
    Tiled,
}

pub fn select_layout(target: &str, layout: LayoutPreset) -> Result<(), TmuxError>

- Validate target as a window target
- Map to tmux select-layout with the appropriate preset name

## Acceptance criteria
- Window layouts can be set to any preset via a domain operation
- The layout presets are modeled as a domain enum
- All three API layers can expose this operation
