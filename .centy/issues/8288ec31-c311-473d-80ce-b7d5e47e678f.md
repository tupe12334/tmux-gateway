# Release workflow does not run tests before building binaries

The release workflow (.github/workflows/release.yml) builds binaries for 4 targets (x86_64-linux, aarch64-linux, x86_64-macos, aarch64-macos) and creates a GitHub release, but it NEVER runs tests.

## Problem:

* A tagged release could ship broken code
* The CI workflow only runs on push to main and PRs, not on tags
* release.yml triggers on ‘push: tags: v\*’ which doesn’t trigger ci.yml

## Fix:

Either:

1. Add a ‘test’ job to release.yml that runs before the build job
1. Or make release.yml depend on CI passing by requiring the tag push to also trigger ci.yml (add ‘tags: \[v\*\]’ to ci.yml push trigger)
1. At minimum add ‘cargo test’ to the build step before packaging
