# Add domain pagination support for list operations

list_sessions, list_windows, and list_panes return all results as a single Vec. On systems with many sessions/windows, this can produce large responses with no way to page through results.

## Problem
- list_sessions() returns every session on the system — no limit or offset
- A tmux server with hundreds of sessions produces a proportionally large response
- No way to sort results (by name, creation time, etc.)
- No way to filter results at the domain level (e.g., only attached sessions)
- All three API layers inherit the "return everything" behavior

## What to do
Add query options to list operations:

pub struct ListOptions {
    pub limit: Option<u32>,
    pub offset: Option<u32>,
    pub filter: Option<ListFilter>,
    pub sort_by: Option<SortField>,
}

pub enum ListFilter {
    AttachedOnly,
    DetachedOnly,
    NameContains(String),
}

pub enum SortField {
    Name,
    CreatedAt,
    WindowCount,
}

Apply filtering and pagination after parsing tmux output but before returning to callers. This keeps the domain responsible for query semantics, not API layers.

## Acceptance criteria
- List operations accept optional pagination and filter parameters
- Results can be limited, offset, filtered, and sorted at the domain level
- API layers expose these parameters in their own formats (query params, arguments, fields)
