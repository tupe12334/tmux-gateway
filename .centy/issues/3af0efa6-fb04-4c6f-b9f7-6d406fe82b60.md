# Add deployment and production operations guide

No documentation exists for deploying tmux-gateway in production. Users must figure out reverse proxy setup, resource limits, monitoring, and troubleshooting on their own.

## What to do
- Add a docs/deployment.md covering:
  - Recommended reverse proxy configuration (nginx/caddy) for TLS termination
  - Systemd service file example
  - Resource limits and tuning (max connections, timeouts)
  - Monitoring setup (Prometheus + Grafana dashboard)
  - Log aggregation recommendations
- Add a docs/troubleshooting.md with common issues and solutions
- Add architecture diagram showing component relationships

## Acceptance criteria
- A new user can deploy to production following the guide without external help
- Common failure modes are documented with remediation steps
