# Add TmuxFormat builder for type-safe format string construction

## Problem

Domain operations build tmux format strings as raw string literals scattered across modules:

- `sessions.rs`: `"#{session_name}\t#{session_windows}\t#{session_created}\t#{session_attached}"`
- `list_windows.rs`: `"#{window_index}\t#{window_name}\t#{window_panes}\t#{window_active}"`
- `list_panes.rs`: `"#{pane_id}\t#{pane_width}\t#{pane_height}\t#{pane_active}"`

These are fragile: typos in format variable names compile fine but produce empty/wrong output at runtime. Adding a new field requires updating both the format string and the parser in lockstep.

## Proposed change

Add a `TmuxFormat` builder in `tmux-gateway-core` that makes format string construction type-safe:

```rust
pub enum TmuxVariable {
    SessionName, SessionWindows, SessionCreated, SessionAttached,
    WindowIndex, WindowName, WindowPanes, WindowActive,
    PaneId, PaneWidth, PaneHeight, PaneActive,
    PanePid, PaneCurrentPath, PaneCurrentCommand,
    // extensible for future fields
}

pub struct TmuxFormat {
    variables: Vec<TmuxVariable>,
    separator: char,
}

impl TmuxFormat {
    pub fn new() -> Self;
    pub fn sep(mut self, sep: char) -> Self;
    pub fn field(mut self, var: TmuxVariable) -> Self;

    /// Pure: produce the tmux format string
    pub fn build(&self) -> String;

    /// Pure: parse a single output line according to this format's field order
    pub fn parse_line<'a>(&self, line: &'a str) -> Result<Vec<&'a str>, TmuxError>;
}
```

This couples the format definition to the parsing contract in a single place. Adding a new field to `TmuxSession` means adding a `TmuxVariable` variant and a `.field()` call — the parser automatically expects the right number of columns.

## Why this matters

- Eliminates a class of silent runtime bugs (typo in format variable name)
- Makes adding new fields to domain types safer (single point of change)
- Issue #55 (enrich TmuxPane with path/command) and #58 (stable IDs) both require adding format fields — this builder prevents regressions
- Issue #73 (pane PID) needs `pane_pid` — easy to add as a variant

## Acceptance criteria

- [ ] `TmuxVariable` enum covering all currently-used tmux format variables
- [ ] `TmuxFormat` builder with `field()`, `sep()`, `build()`, `parse_line()`
- [ ] Refactor `sessions.rs`, `list_windows.rs`, `list_panes.rs` to use the builder
- [ ] Unit tests: build produces correct format string, parse_line handles correct/incorrect column counts
- [ ] Round-trip property: build → tmux output → parse_line recovers all fields
