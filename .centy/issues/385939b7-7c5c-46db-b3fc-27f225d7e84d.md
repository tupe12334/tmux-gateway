# Add make test and make check targets to Makefile

## Problem
The Makefile has targets for build, run, clean, and docker but no targets for running tests or the full pre-push check suite. Developers must know the exact cargo commands to run tests, and there's no way to run the same checks locally that the pre-push hook runs.

## Current state
Makefile targets: build, run, clean, schemas, docker-up, docker-down.
Pre-push hook runs: fmt, build, test, clippy, doc, deny, audit, machete, outdated, cspell.

## Proposed change
Add Makefile targets:
```makefile
test:
	cargo test --workspace

lint:
	cargo fmt --all -- --check
	cargo clippy --workspace -- -D warnings

check: lint test
	cargo doc --workspace --no-deps
	cargo deny check
	cargo audit
	cargo machete
	cspell --no-progress "**"
```

`make check` mirrors the pre-push hook, letting developers validate locally before pushing.

## Acceptance criteria
- [ ] `make test` runs workspace tests
- [ ] `make lint` runs fmt check and clippy
- [ ] `make check` runs the full pre-push validation suite
- [ ] README documents these targets
