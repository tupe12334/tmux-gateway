# Add domain operations for tmux session environment variables

## Problem

tmux maintains per-session environment variables that propagate to child processes in panes. These are managed via `show-environment` and `set-environment` commands, which are distinct from session options (covered by #59).

Environment variables are essential for:
- Setting PATH or custom vars for processes in a session
- Configuring application behavior per session (e.g., DATABASE_URL, LOG_LEVEL)
- Automation workflows that need isolated environments per session

The domain currently has no operations for managing session environments.

## tmux commands

- `tmux show-environment -t session` — list all env vars for a session
- `tmux set-environment -t session VAR value` — set an env var
- `tmux set-environment -t session -u VAR` — unset an env var

## Proposed changes

1. Add `TmuxEnvVar` domain type:
```rust
pub struct TmuxEnvVar {
    pub name: String,
    pub value: Option<String>,  // None means inherited/unset marker
}
```

2. Add domain operations:
- `show_environment(session: &str) -> Result<Vec<TmuxEnvVar>, TmuxError>`
- `set_environment(session: &str, name: &str, value: &str) -> Result<(), TmuxError>`
- `unset_environment(session: &str, name: &str) -> Result<(), TmuxError>`

3. Add validation for env var names (alphanumeric + underscore, no injection)
4. Add corresponding TmuxCommands trait methods

## Acceptance criteria

- [ ] TmuxEnvVar domain type exists in core crate
- [ ] show_environment, set_environment, unset_environment are domain functions
- [ ] Env var names are validated for safety
- [ ] Parsing of show-environment output extracted as a pure function
- [ ] TmuxCommands trait includes the new methods
