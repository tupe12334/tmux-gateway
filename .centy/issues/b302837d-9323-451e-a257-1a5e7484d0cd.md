# Add domain operations for tmux paste buffers

## Problem

tmux maintains a set of paste buffers — an internal clipboard system for transferring text between panes, windows, and sessions. The domain currently has no concept of buffers despite them being a core tmux data entity.

Paste buffers enable automation workflows like:
- Extracting text from one pane's output and pasting into another
- Programmatically setting clipboard content for panes to consume
- Building inter-pane communication pipelines

## tmux commands

- `tmux list-buffers` — list all paste buffers with names and sizes
- `tmux show-buffer [-b name]` — get content of a paste buffer
- `tmux set-buffer [-b name] content` — set paste buffer content
- `tmux paste-buffer -t target [-b name]` — paste buffer content into a pane
- `tmux delete-buffer [-b name]` — delete a paste buffer

## Proposed changes

1. Add `TmuxBuffer` domain type:
```rust
pub struct TmuxBuffer {
    pub name: String,
    pub size: u64,
}
```

2. Add domain operations:
- `list_buffers() -> Result<Vec<TmuxBuffer>, TmuxError>`
- `get_buffer(name: Option<&str>) -> Result<String, TmuxError>`
- `set_buffer(name: Option<&str>, content: &str) -> Result<(), TmuxError>`
- `paste_buffer(target: &str, name: Option<&str>) -> Result<(), TmuxError>`
- `delete_buffer(name: &str) -> Result<(), TmuxError>`

3. Add buffer name validation (alphanumeric + underscore)
4. Add corresponding TmuxCommands trait methods
5. Extract list-buffers parsing as a pure function

## Acceptance criteria

- [ ] TmuxBuffer domain type exists in core crate
- [ ] All five buffer operations are domain functions
- [ ] Buffer name validation prevents injection
- [ ] TmuxCommands trait includes buffer methods
- [ ] Parsing extracted as testable pure function
