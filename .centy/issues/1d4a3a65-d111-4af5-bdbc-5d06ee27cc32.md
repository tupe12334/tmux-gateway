# Add typed target value objects (SessionTarget, WindowTarget, PaneTarget)

Currently all target parameters in domain operations are raw `&str` values, validated ad-hoc at each call site. This violates the DDD principle of encoding invariants in the type system.

## Problem
- `send_keys(target: &str, ...)`, `kill_pane(target: &str)`, etc. accept any string
- Validation is scattered: each operation calls `validate_pane_target()` / `validate_window_target()` independently
- Nothing prevents passing a session target where a pane target is expected — the compiler can't help
- Raw strings provide no semantic distinction between target types

## Proposed solution
Introduce value objects that validate on construction and are consumed by domain operations:

```rust
pub struct SessionTarget(String);   // e.g. "my-session"
pub struct WindowTarget(String);    // e.g. "my-session:0"  
pub struct PaneTarget(String);      // e.g. "my-session:0.1"
```

Each type should:
- Validate format in `TryFrom<&str>` / `new()`, returning `ValidationError` on failure
- Implement `AsRef<str>` for passing to tmux_interface
- Be cheaply cloneable (Arc<str> or small-string optimization)
- Provide accessor methods for parts (e.g. `PaneTarget::session()`, `PaneTarget::window()`, `PaneTarget::pane()`)

Domain operations change from `fn kill_pane(target: &str)` to `fn kill_pane(target: &PaneTarget)`, making invalid targets unrepresentable.

## Scope
- Domain layer only — API layers handle string→value-object conversion at the boundary
- Replaces scattered `validate_*_target()` calls with construction-time validation
- All existing domain operations updated to accept typed targets
