# Add respawn-pane and respawn-window domain operations

## Problem

When a process exits in a tmux pane or window, the pane/window remains in a dead state. tmux provides `respawn-pane` and `respawn-window` to restart dead panes/windows with a new command. The domain currently has no operations for this.

Respawn is essential for automation and long-running gateway scenarios:
- Restarting a crashed process without destroying the pane layout
- Running a different command in an existing pane slot
- Building resilient session templates where processes auto-restart

## tmux commands

- `tmux respawn-pane -t target [command]` — restart a dead pane (optionally with a new command)
- `tmux respawn-window -t target [command]` — restart a dead window

Both support `-k` flag to kill an active pane/window before respawning.

## Proposed changes

1. Add domain operations:
```rust
pub async fn respawn_pane(target: &str, command: Option<&str>, kill_existing: bool) -> Result<(), TmuxError>
pub async fn respawn_window(target: &str, command: Option<&str>, kill_existing: bool) -> Result<(), TmuxError>
```

2. Add validation for targets (reuse existing validate_pane_target / validate_window_target)
3. Add command parameter validation (same as #76)
4. Update TmuxCommands trait with new methods
5. Consider adding a `PaneNotRunning` error variant for cases where respawn is called on an active pane without kill_existing

## Acceptance criteria

- [ ] respawn_pane and respawn_window are domain functions
- [ ] Optional command parameter for new process
- [ ] kill_existing flag controls behavior with active panes
- [ ] Target validation applied
- [ ] TmuxCommands trait includes respawn methods
