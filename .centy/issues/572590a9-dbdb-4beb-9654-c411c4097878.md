# Add custom metadata tagging for sessions, windows, and panes

tmux resources (sessions, windows, panes) have no support for custom metadata. In production, teams need to tag resources with environment (staging/prod), owner, purpose, or project. tmux has no native tagging, so the gateway domain must provide this capability.

## Problem

- No way to associate custom key-value metadata with sessions, windows, or panes
- Clients cannot filter or group resources by purpose, environment, or team
- Authorization policies (#113) and resource limits (#114) have no way to scope by tag
- Multi-tenant deployments cannot distinguish resources belonging to different teams
- Session names alone are insufficient for organizational classification

## Solution

Add a domain-level metadata store that associates tags with tmux resource targets:

```rust
pub struct ResourceTag {
    pub key: String,
    pub value: String,
}

pub struct ResourceMetadata {
    pub target: String,           // session name, window target, or pane target
    pub resource_type: ResourceType,
    pub tags: Vec<ResourceTag>,
}

pub enum ResourceType { Session, Window, Pane }

/// Tag a resource with key-value metadata
pub async fn tag_resource(target: &str, resource_type: ResourceType, key: &str, value: &str) -> Result<(), TmuxError>

/// Remove a tag from a resource
pub async fn untag_resource(target: &str, key: &str) -> Result<(), TmuxError>

/// List tags for a resource
pub async fn get_tags(target: &str) -> Result<Vec<ResourceTag>, TmuxError>

/// Find resources matching tag filters
pub async fn find_by_tags(filters: &[ResourceTag]) -> Result<Vec<ResourceMetadata>, TmuxError>
```

The metadata store can be backed by tmux environment variables (`set-environment`) or an in-memory/file-based store managed by the domain.

## Impact

- Enables organizational resource management through the gateway
- Foundation for tag-based authorization policies and resource limits
- Supports multi-tenant and multi-environment deployments
- API-agnostic — all transport layers expose the same tagging operations
