# Add optional TLS support for HTTP and gRPC

Both servers bind on plaintext TCP. For production deployments, TLS is needed.

## Implementation
- Add axum-server with rustls for HTTPS
- Add tonic TLS configuration for gRPC
- Environment variables: TLS_CERT_PATH, TLS_KEY_PATH, TLS_ENABLED (default: false)
- When TLS_ENABLED=true, require cert and key paths
- Update port_table to show HTTPS/gRPCS instead of HTTP/gRPC when TLS is active
- Document TLS setup in README
- Consider optional mTLS for client certificate authentication
