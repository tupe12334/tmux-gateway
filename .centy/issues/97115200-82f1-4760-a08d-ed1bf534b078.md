# Add graceful shutdown with signal handling

The server currently joins two tokio tasks but has no graceful shutdown handling. When receiving SIGTERM/SIGINT, connections may be dropped mid-request.

## Requirements

* Handle SIGTERM and SIGINT signals via tokio::signal
* Gracefully drain in-flight HTTP and gRPC requests
* Set a configurable shutdown timeout (e.g. 30s default)
* Log shutdown progress (received signal, draining, done)
* Axum supports graceful shutdown via Server::with_graceful_shutdown
* Tonic supports it via Server::serve_with_shutdown

## Implementation

* Use tokio::signal::ctrl_c() and unix::signal(SignalKind::terminate())
* Share a shutdown notification channel between both servers
* Add SHUTDOWN_TIMEOUT_SECS env var (default 30)
