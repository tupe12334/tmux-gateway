# Add request body size limits to prevent DoS

The Axum HTTP server has no body size limits configured. An attacker could send arbitrarily large request bodies to exhaust server memory.

## Fix
- Add axum::extract::DefaultBodyLimit middleware to the router
- Set a reasonable default (e.g. 1MB for JSON APIs)
- Make configurable via MAX_REQUEST_BODY_BYTES env var
- Apply to both REST and GraphQL endpoints
