# Add kill_server domain operation to shut down the tmux server

The domain can kill individual sessions, windows, and panes but has no operation to shut down the entire tmux server. This is a core tmux lifecycle operation needed for clean system management.

## Problem
- No way to cleanly shut down all tmux sessions and the server process through the gateway
- Killing sessions one-by-one is racy — new sessions can be created between kills
- tmux kill-server is the canonical way to do a full teardown
- Operators must SSH in and run tmux kill-server manually

## What to do
Add to tmux-gateway-core:

pub fn kill_server() -> Result<(), TmuxError>

- Maps to tmux kill-server
- Returns TmuxNotRunning if no server is running (not an error — idempotent)
- This is a destructive operation — API layers should require confirmation or elevated permissions

## Acceptance criteria
- The tmux server can be shut down via a domain operation
- The operation is idempotent (no error if server already stopped)
- All three API layers can expose this operation
