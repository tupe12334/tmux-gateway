# Add domain operations for tmux session options

## Problem
Tmux supports per-session and per-window options (e.g., mouse mode, status bar, history limit) via `set-option` and `show-options`. The domain layer has no concept of tmux options, preventing clients from configuring session behavior programmatically.

## Proposed change
Add to tmux-gateway-core:

```rust
pub struct TmuxOption {
    pub name: String,
    pub value: String,
    pub scope: OptionScope,
}

pub enum OptionScope {
    Global,
    Session,
    Window,
}

pub async fn get_option(target: &str, name: &str, scope: OptionScope) -> Result<String, TmuxError>
pub async fn set_option(target: &str, name: &str, value: &str, scope: OptionScope) -> Result<(), TmuxError>
pub async fn list_options(target: &str, scope: OptionScope) -> Result<Vec<TmuxOption>, TmuxError>
```

Common use cases: setting `mouse on`, adjusting `history-limit`, configuring `status-left`/`status-right`.

## Acceptance criteria
- [ ] TmuxOption and OptionScope types in core crate
- [ ] get_option, set_option, list_options functions
- [ ] Input validation on option names (prevent injection)
- [ ] Added to TmuxCommands trait
- [ ] Available from all API layers
