# Add new_session options for start directory and initial command

new_session only accepts a session name. Tmux new-session supports starting in a specific directory (-c) and running an initial command as the shell. These are essential for automation workflows.

## Problem
- new_session(name) creates a session in the current working directory with the default shell
- No way to specify a start directory (tmux -c flag)
- No way to run a command in the new session (e.g., new-session -d -s build "make all")
- Automation that needs sessions in specific directories must send_keys("cd /path") after creation, which is racy

## What to do
Extend new_session to accept options:

pub struct NewSessionOptions {
    pub name: String,
    pub start_directory: Option<String>,
    pub shell_command: Option<String>,
    pub window_name: Option<String>,
}

pub fn new_session(options: NewSessionOptions) -> Result<String, TmuxError>

- Validate start_directory exists if provided
- Validate shell_command doesn't contain injection vectors
- Map to tmux new-session -d -s name [-c dir] [-n window_name] [shell_command]

## Acceptance criteria
- Sessions can be created with a specific working directory
- Sessions can be created with an initial command
- Validation prevents path traversal and command injection
- All API layers can expose these options
