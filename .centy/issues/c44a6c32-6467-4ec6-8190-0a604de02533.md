# Add WindowLayout value object for type-safe layout management

## Problem

Tmux window layouts are represented as opaque strings (e.g., `"34ed,80x24,0,0,0"` or `"bb62,159x48,0,0{79x48,0,0,80x48,80,0}"`). The domain layer has no structured representation for these, making it impossible to validate, compare, or programmatically construct layouts.

Issue #69 covers layout management operations (select-layout, etc.) but there is no domain type to represent a layout itself.

## Proposed change

Add a `WindowLayout` value object in `tmux-gateway-core` that:

1. **Parses** tmux layout strings into a structured tree:
   ```rust
   pub enum LayoutNode {
       Leaf { width: u32, height: u32, x: u32, y: u32, pane_id: u32 },
       Horizontal { width: u32, height: u32, x: u32, y: u32, children: Vec<LayoutNode> },
       Vertical { width: u32, height: u32, x: u32, y: u32, children: Vec<LayoutNode> },
   }

   pub struct WindowLayout {
       checksum: String,
       root: LayoutNode,
   }
   ```

2. **Validates** layout structure (dimensions must be consistent, no overlapping panes)

3. **Serializes** back to tmux layout string format for `select-layout` operations

4. **Derives** `PartialEq`, `Eq`, `Clone`, `Debug` for comparison and testing

5. **Provides** pure helper methods:
   - `pane_count() -> usize`
   - `dimensions() -> (u32, u32)`
   - `contains_pane(id: u32) -> bool`
   - `is_tiled() -> bool` (all panes same size)

## Why this matters

- Layout operations (#69) need a domain type to operate on, not raw strings
- Snapshot/restore (#107) needs to serialize and deserialize layouts
- Layout comparison enables change detection (is the layout different after a split?)

## Acceptance criteria

- [ ] `WindowLayout` and `LayoutNode` types in `tmux-gateway-core`
- [ ] Pure `parse()` function: `&str -> Result<WindowLayout, TmuxError>`
- [ ] Pure `to_tmux_string()` method: `&WindowLayout -> String`
- [ ] Round-trip property: `parse(layout.to_tmux_string()) == layout`
- [ ] Unit tests for common layout patterns (single pane, even-horizontal, even-vertical, tiled, nested)
- [ ] Unit tests for invalid layout strings
