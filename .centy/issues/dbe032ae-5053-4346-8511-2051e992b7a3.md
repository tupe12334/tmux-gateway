# Add kill-server domain operation for full tmux server shutdown

The domain has operations to kill individual sessions, windows, and panes, but no operation to shut down the entire tmux server. The tmux `kill-server` command is an essential administrative operation for gateway management.

## Problem
- No way to gracefully shut down the tmux server through the gateway
- Operators managing tmux-gateway in production may need to reset the tmux server
- The domain model is incomplete without this top-level lifecycle operation
- kill-session exists but killing all sessions one-by-one is not equivalent to kill-server (race conditions, zombie processes)

## Proposed solution
Add a `kill_server` domain operation:

\`\`\`rust
/// Shut down the tmux server, destroying all sessions.
/// Returns Ok(()) if the server was killed or was already not running.
pub async fn kill_server() -> Result<(), TmuxError>
\`\`\`

Design considerations:
- Should be idempotent: if the server is already down, return Ok(()) rather than an error
- Uses tmux \`kill-server\` command under the hood
- No target validation needed (no parameters)
- This is a destructive operation — API layers should gate it appropriately (e.g. require confirmation header, admin role)

## Scope
- New \`kill_server.rs\` module in \`tmux-gateway-core\`
- Add to \`TmuxCommands\` trait
- Export from \`lib.rs\`
