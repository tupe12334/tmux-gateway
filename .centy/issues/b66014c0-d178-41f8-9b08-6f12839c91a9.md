# Add input validation and sanitization

No validation exists on user-provided inputs (session names, targets). This could lead to tmux command injection or confusing errors.

## Required Validations
- Session names: alphanumeric + hyphens/underscores, no spaces, reasonable length limit
- Kill targets: validate format (session, session:window, session:window.pane)
- Reject empty strings for all required fields
- Validate target exists before attempting kill operations (provide clear 404)

## Implementation
- Add validation functions in tmux-gateway-core
- Return InvalidTarget or InvalidInput errors (ties into custom error types issue)
- Apply validation consistently across REST, GraphQL, and gRPC layers
- Add unit tests for all validation cases
