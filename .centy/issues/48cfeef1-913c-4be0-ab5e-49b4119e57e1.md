# Add /version endpoint with build metadata

No endpoint exposes version or build info. This makes debugging deployed instances difficult.

## What to do
- Add a GET /version REST endpoint returning: version (from Cargo.toml), git commit hash, build timestamp
- Inject build metadata via a build.rs script using env vars (VERGEN or manual)
- Include version info in gRPC server reflection metadata
- Add the version to the startup log output

## Acceptance criteria
- GET /version returns JSON with version, commit, and build_time fields
- GraphQL query { version } returns the same data
- Build metadata is injected at compile time, not runtime
