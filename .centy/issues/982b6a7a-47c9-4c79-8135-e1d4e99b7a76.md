# Add capture options to capture_pane for scroll history and line ranges

## Problem

The capture_pane domain operation only captures the visible pane content with no options. Tmux’s capture-pane supports useful options like scroll-back history (-S start_line), end line (-E end_line), and escape sequences (-e). These are domain-level concerns about what content to capture, not transport concerns.

## Current state

````rust
pub async fn capture_pane(target: &str) -> Result<String, TmuxError>
````

Uses `CapturePane::new().stdout().target_pane(...)` with no options.

## Proposed change

Add a domain-level options type:

````rust
pub struct CaptureOptions {
    pub start_line: Option<i32>,   // -S flag, negative for history
    pub end_line: Option<i32>,     // -E flag
    pub escape_sequences: bool,     // -e flag
}

pub async fn capture_pane(target: &str) -> Result<String, TmuxError>
pub async fn capture_pane_with_options(target: &str, opts: &CaptureOptions) -> Result<String, TmuxError>
````

Keep the existing simple capture_pane as a convenience wrapper. The options type is a pure domain concept (what to capture), not a transport concern.
