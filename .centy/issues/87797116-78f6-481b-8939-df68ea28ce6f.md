# Include stable tmux IDs in TmuxSession and TmuxWindow domain models

## Problem

TmuxSession and TmuxWindow are identified only by name, but tmux assigns stable unique IDs to sessions (`$0`, `$1`, …) and windows (`@0`, `@1`, …) that persist even when sessions/windows are renamed. The current domain models don't capture these IDs, meaning a rename operation can break references held by clients.

## Current state

````rust
pub struct TmuxSession {
    pub name: String,        // can change via rename
    pub windows: u32,
    pub created: String,
    pub attached: bool,
    // no stable ID
}

pub struct TmuxWindow {
    pub index: u32,          // can change when windows are reordered
    pub name: String,        // can change via rename
    pub panes: u32,
    pub active: bool,
    // no stable ID
}
````

Tmux provides `#{session_id}` (e.g., "$0") and `#{window_id}` (e.g., "@3") format variables that are stable identifiers.

## Proposed change

Add ID fields to domain models:

````rust
pub struct TmuxSession {
    pub id: String,          // "$0" — stable across renames
    pub name: String,
    ...
}

pub struct TmuxWindow {
    pub id: String,          // "@3" — stable across renames/reorder
    pub index: u32,
    pub name: String,
    ...
}
````

Update format strings in list operations to include `#{session_id}` and `#{window_id}`. Domain operations that accept targets should accept either name or ID.
