# Enrich domain entities with proper types (DateTime, NonZeroU32) instead of raw primitives

Domain structs use raw primitives where richer types would better express domain semantics and prevent invalid states.

## Problem
- TmuxSession.created is a String — should be a DateTime
- TmuxSession.windows is u32 — a session always has at least 1 window (NonZeroU32)
- TmuxWindow.panes is u32 — a window always has at least 1 pane (NonZeroU32)
- TmuxPane.id is String — should be a validated PaneId value object
- No domain methods on entities (e.g., session.is_attached(), window.is_active())

## What to do
- Parse created timestamp into chrono::DateTime<Utc> during list_sessions (keep String as fallback if parsing fails)
- Use NonZeroU32 or a WindowCount newtype for windows/panes count
- Add domain methods where they express business meaning
- API layers handle serialization (DateTime -> ISO 8601 string, NonZeroU32 -> u32)

## Acceptance criteria
- Domain types prevent representing invalid states (0 windows, unparseable dates)
- API layers convert to wire-format types, domain layer uses rich types
- Parsing errors in list operations are surfaced rather than silently defaulting to 0
