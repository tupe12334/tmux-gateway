# Add find_session domain query to look up a session by name

There is no way to query a single session by name. The only option is list_sessions() and filtering client-side. This is inefficient and creates a TOCTOU race for operations that need to verify existence before acting.

## Problem
- Checking if a session exists requires calling list_sessions() and iterating through all results
- No tmux has-session equivalent in the domain layer
- Operations like "create if not exists" or "rename only if target exists" require two round-trips with a race window
- Issue #39 mentions "session existence check" but doesn't specify the query interface

## What to do
Add to tmux-gateway-core:

pub async fn has_session(name: &str) -> Result<bool, TmuxError>
pub async fn get_session(name: &str) -> Result<Option<TmuxSession>, TmuxError>

- has_session maps to tmux has-session -t (returns bool based on exit code)
- get_session uses tmux display-message with format strings to get a single session's details
- Both validate the name parameter

## Acceptance criteria
- A single session can be queried by name without listing all sessions
- The query returns None/false for non-existent sessions rather than an error
- All three API layers can expose single-session lookup
