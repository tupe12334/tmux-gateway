# Introduce SplitDirection enum to replace raw bool in split_window

split_window takes a raw bool for horizontal/vertical direction. This has unclear semantics at the call site — callers must remember whether true means horizontal or vertical.

## Problem
- split_window(target, horizontal: bool) — does true mean split horizontally or split into horizontal panes?
- API layers pass a bool with no domain meaning
- tmux itself distinguishes -h (horizontal split) and -v (vertical split) — the domain should model this explicitly

## What to do
Define a domain enum in the core crate:

pub enum SplitDirection {
    Horizontal,
    Vertical,
}

Change the function signature:
- split_window(target: &str, direction: SplitDirection)

API layers deserialize into SplitDirection from their request formats (bool, string, enum).

## Acceptance criteria
- No raw bool parameter for split direction in the domain layer
- Call sites read clearly: split_window(target, SplitDirection::Horizontal)
- API layers convert their representation to the domain enum
