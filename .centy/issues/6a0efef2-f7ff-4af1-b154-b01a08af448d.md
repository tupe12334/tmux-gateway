# Add select_window and select_pane domain operations

## Problem

Tmux supports selecting (focusing) a window or pane with `select-window` and `select-pane`, but the domain layer has no operations for these. For remote tmux control scenarios, being able to focus a specific window or pane is essential — it determines what the user sees when they attach to the session.

## Proposed change

Add to tmux-gateway-core:

````rust
pub async fn select_window(target: &str) -> Result<(), TmuxError>
pub async fn select_pane(target: &str) -> Result<(), TmuxError>
````

These use the existing target validation (validate_window_target, validate_pane_target) and follow the same pattern as other domain operations.

* [ ] select_window and select_pane functions added to core
* [ ] Target validation applied
