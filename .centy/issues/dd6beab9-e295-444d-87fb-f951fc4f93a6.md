# Add unit tests for TmuxError::from_stderr classification logic

## Problem
TmuxError::from_stderr() is a critical pure function that classifies raw tmux stderr output into domain error variants (SessionNotFound, WindowNotFound, PaneNotFound, TmuxNotRunning, CommandFailed). It has zero unit tests despite being core error classification logic.

This is exactly the kind of functional core logic that should have comprehensive tests — it's a pure function with no I/O dependencies that can be tested trivially.

## Current state
In error.rs, from_stderr() does case-insensitive string matching on stderr patterns:
- "no server running" → TmuxNotRunning
- "session not found" / "can't find session" → SessionNotFound
- "window not found" / "can't find window" → WindowNotFound
- "pane not found" / "can't find pane" → PaneNotFound
- anything else → CommandFailed

None of these branches have test coverage.

## Proposed change
Add #[cfg(test)] tests in error.rs covering:
- Each known stderr pattern maps to the correct variant
- Case variations (uppercase, mixed case)
- Multiple patterns per variant (e.g., both "session not found" and "can't find session")
- Unknown stderr falls through to CommandFailed
- The target parameter is correctly passed into the error variant
- Edge cases: empty stderr, stderr with extra whitespace

## Acceptance criteria
- [ ] Test for each error variant mapping
- [ ] Test for fallthrough to CommandFailed
- [ ] Test that target parameter is preserved in error variants
- [ ] All tests are pure (no tmux dependency)
