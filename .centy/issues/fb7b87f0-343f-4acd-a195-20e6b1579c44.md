# Add split_window options for start directory, command, and target size

split_window only accepts a target and direction. Tmux split-window also supports start directory, initial command, and size specification. These are needed for programmatic pane creation workflows.

## Problem
- split_window(target, horizontal) creates a pane with default shell and even split
- No way to specify the new pane's size (-l lines/percentage or -p percentage)
- No way to set start directory (-c) for the new pane
- No way to run a command in the new pane
- Building a multi-pane layout requires split + send_keys for each pane, which is racy

## What to do
Extend split_window to accept options:

pub struct SplitWindowOptions {
    pub target: String,
    pub direction: SplitDirection,    // from #58
    pub size: Option<PaneSize>,
    pub start_directory: Option<String>,
    pub shell_command: Option<String>,
}

pub enum PaneSize {
    Lines(u32),
    Percentage(u8),  // 1-99
}

pub fn split_window(options: SplitWindowOptions) -> Result<(), TmuxError>

## Acceptance criteria
- Pane splits can specify size, directory, and command
- PaneSize percentage is validated to 1-99 range
- Consistent options pattern with new_session (#79) and new_window (#84)
