# Add display-popup domain operation for overlay windows

tmux 3.2+ introduced display-popup, which creates overlay popup windows within a pane. Popups are useful for contextual menus, quick command execution, and interactive prompts. No existing issue covers this capability.

## Problem

- display-popup is a major tmux feature (3.2+) with no domain representation
- Popups enable rich UI experiences through the gateway (contextual forms, command palettes, confirmations)
- No way to programmatically create, configure, or close popup windows
- Complements display-message (#98) with interactive overlay capabilities

## Solution

Add domain operations for popup management:

```rust
pub struct PopupOptions {
    pub width: Option<u32>,      // Popup width (columns or percentage)
    pub height: Option<u32>,     // Popup height (rows or percentage)
    pub command: Option<String>, // Shell command to run in popup
    pub style: Option<String>,   // Border style
    pub title: Option<String>,   // Popup title
    pub close_on_exit: bool,     // Close popup when command exits
}
```
