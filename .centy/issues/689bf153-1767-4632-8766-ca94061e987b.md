# Sync package.json version with Cargo.toml and remove unused fields

package.json has version \"1.0.0\" while Cargo.toml has \"0.1.0\". It also contains unused Node.js fields (main, type) since the project is Rust-only. The package.json exists solely for husky git hooks.

## What to do
- Remove unused fields: main, type, keywords, author, license (already in Cargo.toml)
- Either sync version with Cargo.toml or remove version field entirely
- Add a comment explaining that package.json exists only for husky hooks
- Consider whether husky can be replaced with a Rust-native pre-commit solution (e.g., cargo-husky)

## Acceptance criteria

- Version is consistent across all project files
