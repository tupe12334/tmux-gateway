# Pin Rust toolchain version for reproducible builds

No rust-toolchain.toml exists. Different developers and CI may use different Rust versions, leading to inconsistent behavior, especially with edition = "2024" features.

## What to do
- Add a rust-toolchain.toml with channel, components (rustfmt, clippy), and targets
- Pin to a specific stable version (e.g., 1.85.0 or latest stable that supports edition 2024)
- Update CI to use the pinned version instead of @stable
- Document the minimum supported Rust version (MSRV) in Cargo.toml

## Acceptance criteria
- All developers and CI use the same Rust version
- cargo build works consistently across environments
- MSRV is documented in Cargo.toml via rust-version field
