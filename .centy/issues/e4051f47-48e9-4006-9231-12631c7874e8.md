# Add rotate-window domain operation for cyclic pane reordering

tmux \`rotate-window\` cyclically rotates the positions of panes within a window. Combined with swap-pane (#71) and layout management (#69), this completes the pane arrangement toolkit.

## Problem
- No way to rotate pane positions through the gateway
- Swap (#71) moves individual panes; rotate shifts all panes cyclically
- Layout management (#69) sets predefined layouts; rotate works within any layout
- Missing this leaves a gap in pane arrangement operations

## Proposed solution

\`\`\`rust
/// Rotate pane positions in a window.
/// \`upward\` = true rotates up (each pane takes the previous pane's position),
/// \`upward\` = false rotates down (each pane takes the next pane's position).
pub async fn rotate_window(
    target: &str,    // window target (e.g. "session:0")
    upward: bool,
) -> Result<(), TmuxError>
\`\`\`

- Validates window target
- Uses tmux \`rotate-window -t target [-U | -D]\`
- Maps errors through \`TmuxError::from_stderr\`

## Scope
- New \`rotate_window.rs\` module in \`tmux-gateway-core\`
- Add to \`TmuxCommands\` trait
- Complements #69 (layouts) and #71 (swap/move)
