# Add copy-mode domain operations for programmatic scrollback interaction

tmux copy mode allows navigating and searching the scrollback buffer. While capture_pane (#51) reads content, there are no domain operations for programmatically entering copy mode, scrolling, or searching within it. These operations are essential for automated terminal interaction workflows.

## Problem

- No domain operation to enter or exit copy mode
- No way to programmatically scroll through pane history
- No domain operation to search within a pane's scrollback buffer
- find-window (#86) searches across panes, but there's no intra-pane search
- Automated testing and monitoring workflows need scrollback navigation

## Solution

Add domain operations for copy-mode interaction:

```rust
/// Enter copy mode for a target pane
pub async fn enter_copy_mode(target: &str) -> Result<(), TmuxError> {
    validate_pane_target(target)?;
    // Execute: tmux copy-mode -t target
}

/// Exit copy mode for a target pane
pub async fn cancel_copy_mode(target: &str) -> Result<(), TmuxError> {
    validate_pane_target(target)?;
    // Execute: tmux send-keys -t target -X cancel
}

/// Scroll in copy mode
pub enum ScrollDirection { Up, Down, HalfPageUp, HalfPageDown, Top, Bottom }

pub async fn scroll_pane(

    direction: ScrollDirection,
    lines: Option<u32>,
```

## Impact

- Enables programmatic scrollback navigation and search
- Complements capture_pane for interactive terminal automation
- Useful for automated testing frameworks that need to verify scrollback content
- Fills a gap in pane interaction capabilities
