# Add AnyTarget union type for operations that accept multiple target formats

## Problem

Most domain operations accept a specific target format: `kill_session` takes a session target, `kill_window` takes `session:window`, `kill_pane` takes `session:window.pane`. But some operations can accept ANY target format:

- `capture_pane` can target a session (captures active window's active pane), a window (captures active pane), or a specific pane
- Future operations like `display-message` (#98) and `send-keys` can work at any level

Currently `capture_pane` accepts raw `&str` with no validation (issue #83 addresses validation but not the type). Issue #82 proposes separate typed targets (`SessionTarget`, `WindowTarget`, `PaneTarget`), but there's no union type for operations that accept any of them.

## Proposed change

Add an `AnyTarget` enum in `tmux-gateway-core`:

```rust
#[derive(Clone, Debug, PartialEq, Eq)]
pub enum AnyTarget {
    Session(String),
    Window(String),   // "session:window"
    Pane(String),     // "session:window.pane"
}

impl AnyTarget {
    /// Pure: parse a target string, detecting its format automatically
    pub fn parse(target: &str) -> Result<Self, ValidationError> {
        if target.contains('.') {
            validate_pane_target(target)?;
            Ok(AnyTarget::Pane(target.to_string()))
        } else if target.contains(':') {
            validate_window_target(target)?;
            Ok(AnyTarget::Window(target.to_string()))
        } else {
            validate_session_target(target)?;
            Ok(AnyTarget::Session(target.to_string()))
        }
    }

    /// Pure: get the session name from any target type
    pub fn session_name(&self) -> &str;

    /// Pure: get the tmux-compatible target string
    pub fn as_tmux_arg(&self) -> &str;

    /// Pure: check target specificity level
    pub fn specificity(&self) -> TargetSpecificity;
}

#[derive(Clone, Copy, Debug, PartialEq, Eq, PartialOrd, Ord)]
pub enum TargetSpecificity {
    Session,
    Window,
    Pane,
}
```

Update `capture_pane` to accept `AnyTarget` instead of raw `&str`. Future operations (#98, #106) can also use `AnyTarget`.

## Why this matters

- Provides type safety for operations that accept flexible targets
- `AnyTarget::parse()` validates AND classifies in one step â€” no ambiguity
- Complements issue #82 (separate typed targets) with a union for flexible operations
- `session_name()` extractor enables cache invalidation (issue #139) regardless of target type
- All logic is pure â€” parsing, validation, extraction are zero-I/O
