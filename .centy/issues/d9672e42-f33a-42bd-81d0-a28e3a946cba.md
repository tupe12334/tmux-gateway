# Port default values inconsistent across .env.example, .env.dev, and Dockerfile

## Problem
The project has three different default port configurations that don't match:

- **.env.example**: HTTP_PORT=8080, GRPC_PORT=50051
- **.env.dev**: HTTP_PORT=3000, GRPC_PORT=50051
- **Dockerfile**: HTTP_PORT=3020, GRPC_PORT=50251
- **docker-compose.yml**: defaults to 3020/50251

The README also says HTTP is on 8080 and gRPC on 50051.

This causes confusion when switching between local development, Docker, and following documentation. A developer using `.env.example` as a starting point will have different ports than someone running Docker.

Note: Issue #27 covers the Dockerfile vs .env.example mismatch but doesn't address the .env.dev inconsistency or the three-way disagreement.

## Proposed change
Align all configuration sources on a single set of default ports. Recommended:
- HTTP_PORT=8080 (conventional, matches README)
- GRPC_PORT=50051 (conventional gRPC port)

Update .env.dev, Dockerfile, docker-compose.yml, and README to use these defaults consistently.

## Acceptance criteria
- [ ] .env.example, .env.dev, Dockerfile, docker-compose.yml all use the same default ports
- [ ] README architecture table matches the actual defaults
- [ ] EXPOSE directive in Dockerfile matches ENV defaults
