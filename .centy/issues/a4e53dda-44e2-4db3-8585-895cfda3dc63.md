# Add window and pane navigation domain operations

tmux provides navigation commands (\`last-window\`, \`next-window\`, \`previous-window\`, \`last-pane\`, \`next-layout\`) for moving between windows and panes in a session. Issue #54 covers \`select-window\` and \`select-pane\` for absolute navigation, but relative navigation commands are missing.

## Problem
- No way to move to the next/previous window in a session
- No way to toggle back to the last active window or pane
- These are the most common navigation patterns in tmux usage
- Orchestration scripts often need "go to next window" without knowing the index

## Proposed solution

\`\`\`rust
/// Switch to the last (most recently active) window in a session.
pub async fn last_window(session: &str) -> Result<(), TmuxError>

/// Switch to the next window in a session.
pub async fn next_window(session: &str) -> Result<(), TmuxError>

/// Switch to the previous window in a session.
pub async fn previous_window(session: &str) -> Result<(), TmuxError>

/// Switch to the last (most recently active) pane in a window.
pub async fn last_pane(target: &str) -> Result<(), TmuxError>
\`\`\`

All operations validate their targets and map errors through \`TmuxError::from_stderr\`.

## Scope
- New \`navigation.rs\` module in \`tmux-gateway-core\`
- Add all four operations to \`TmuxCommands\` trait
- Complements #54 (select-window/select-pane) for complete navigation support
