# Add detach-client domain operation to disconnect attached terminals

Issue #83 adds list-clients to query connected terminals, but there is no complementary operation to detach (disconnect) a client. The tmux \`detach-client\` command is essential for managing which terminals are connected to sessions.

## Problem
- No way to programmatically disconnect a client from a session
- Cannot force-detach stale or unwanted connections
- The list-clients + detach-client pair forms a complete client management capability
- Without detach, list-clients is query-only with no ability to act on the results

## Proposed solution
Add domain operations for client detachment:

\`\`\`rust
/// Detach a specific client by its TTY path.
pub async fn detach_client(tty: &str) -> Result<(), TmuxError>

/// Detach all clients from a given session.
pub async fn detach_all_clients(session: &str) -> Result<(), TmuxError>
\`\`\`

- \`detach_client\` targets a specific terminal (e.g. "/dev/ttys003") using \`detach-client -t\`
- \`detach_all_clients\` detaches everyone from a session using \`detach-client -s\`
- Validate TTY path format (must start with /dev/) and session target
- Map "no client" errors to a clear domain error

## Scope
- New \`detach_client.rs\` module in \`tmux-gateway-core\`
- Add to \`TmuxCommands\` trait
- Depends on #83 for the TmuxClient model
- Export from \`lib.rs\`
