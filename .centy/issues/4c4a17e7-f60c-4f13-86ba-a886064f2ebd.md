# CI workflow missing cargo fmt, deny, audit, and cspell checks

The GitHub Actions CI (.github/workflows/ci.yml) only runs build, test, clippy, schema-drift, and docker-build. But the pre-push hook runs additional checks that CI doesn’t:

## Missing from CI:

* cargo fmt –check (formatting)
* cargo deny check (license/advisory/duplicate checks)
* cargo audit (security vulnerabilities)
* cargo machete (unused dependencies)
* cspell (spelling)

## Impact:

* External contributors (who don’t have the pre-push hook) can merge PRs with formatting issues, spelling errors, or advisory violations
* CI gives a false sense of confidence since it’s less strict than local checks

## Fix:

Add jobs to ci.yml for each missing check, or at minimum:

* cargo fmt –check
* cargo deny check
* cargo audit
