# Add Prometheus metrics endpoint for production observability

The project has structured logging but no quantitative metrics. Production deployments need metrics for monitoring, alerting, and capacity planning.

## What to do
- Add a GET /metrics endpoint exposing Prometheus-format metrics
- Track: request count per endpoint, request latency histogram, error count by type, active tmux sessions gauge
- Use the metrics or prometheus crate
- Ensure metrics don't add significant overhead to the hot path

## Key metrics
- tmux_gateway_requests_total{method, endpoint, status}
- tmux_gateway_request_duration_seconds{method, endpoint}
- tmux_gateway_tmux_sessions_total (gauge)
- tmux_gateway_errors_total{error_type}

## Acceptance criteria
- /metrics returns valid Prometheus exposition format
- Metrics update on each request
- Minimal performance impact (<1ms per request)
