# Add session group domain support for shared-window session clusters

tmux supports session groups — multiple sessions that share the same set of windows. This is a powerful feature for collaborative workflows (multiple users viewing the same terminal) and multi-view setups (same windows, different active window selection). The domain model has no representation for this concept.

## Problem
- \`new-session -t existing-session\` creates a grouped session, but the domain has no way to express this
- No domain model for session group relationships
- Cannot query which sessions belong to the same group
- Missing a core tmux concept that enables collaboration and multi-view patterns

## Proposed solution
Add session group support to the domain:

\`\`\`rust
/// Create a new session grouped with an existing session (shared windows).
pub async fn new_grouped_session(
    name: &str,
    group_target: &str, // existing session to group with
) -> Result<String, TmuxError>
\`\`\`

Enrich \`TmuxSession\` model with group information:
\`\`\`rust
pub struct TmuxSession {
    pub name: String,
    pub windows: u32,
    pub created: i64,
    pub attached: bool,
    pub group: Option<String>,      // session group name, if grouped
    pub group_size: Option<u32>,    // number of sessions in the group
}
\`\`\`

Parse \`#{session_group}\` and \`#{session_group_size}\` tmux format variables in list_sessions.

## Scope
- Extend \`new_session\` or add \`new_grouped_session\` in domain layer
- Enrich TmuxSession model with group fields
- Update list_sessions format string to include group info
- Add to TmuxCommands trait
