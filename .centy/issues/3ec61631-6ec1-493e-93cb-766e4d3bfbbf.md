# Add process identity fields to TmuxPane domain model

## Problem

The TmuxPane domain model (`crates/tmux-gateway-core/src/list_panes.rs`) currently has id, width, height, and active fields. Issue #55 covers adding current_path and current_command. However, the pane's process ID (pid) is also a fundamental piece of domain information.

Knowing the pid enables:
- Detecting whether a process is still running in a pane
- Correlating pane activity with system processes
- Automation workflows that need to wait for a process to finish

## Current state

```rust
pub struct TmuxPane {
    pub id: String,
    pub width: u32,
    pub height: u32,
    pub active: bool,
}
```

tmux format variable `#{pane_pid}` provides this data.

## Proposed changes

- Add `pid: u32` field to TmuxPane
- Update the tmux format string in `list_panes()` to include `#{pane_pid}`
- Update the parsing logic to extract pid
- This is a domain model enrichment, independent of which API layer consumes it

## Acceptance criteria

- [ ] TmuxPane has a `pid: u32` field
- [ ] list_panes output includes pane pid
- [ ] All three API representations (REST, GraphQL, gRPC) expose the pid
- [ ] Unit tests for parsing panes with pid field
